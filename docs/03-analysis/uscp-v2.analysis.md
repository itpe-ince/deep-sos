# USCP V2.0 PDCA Check — 정합성 분석 보고서

> **Summary**: Plan + Design vs V2.0 산출물 3종(견적서·사업계획서·RFI 답변서)의 정합성을 두 에이전트(gap-detector + design-validator) 병렬 호출로 종합 분석.

> **Design Doc**: `docs/02-design/features/uscp-v2.design.md`
> **검증 일시**: 2026-05-14

## 1. 종합 Match Rate

| 평가자 | 점수 | 평가 |
|--------|:---:|------|
| **gap-detector** (Plan + Design vs V2.0 산출물 충족도) | **97%** | 목표 95+ 달성 ✅ |
| **design-validator** (V2.0 산출물 내부 무결성·상호 일관성) | **88%** | 목표 95+ 미달 ⚠️ |

### 1.1 평가 격차 원인 분석

두 에이전트의 점수 격차(97% vs 88%)는 검증 관점 차이에서 발생:

- **gap-detector**: 본질적 사업 결정·범위·견적의 충족도 평가 → 모든 핵심 결정 사항이 V2.0에 반영되었으므로 높은 점수
- **design-validator**: 표면 표기 정확성(카운트·산식·신규 항목 표기)까지 엄격 평가 → 카운트 표기 불일치·AUTH-12 누락 등 5건 발견

**결론**: 사업 본질·범위·견적은 양호하나 **표면 표기 오류 5건이 발견**되어 Critical/Major 수정이 필요. 수정 후 종합 점수 95+ 달성 가능.

## 3. 발견된 이슈 목록 (design-validator 결과)

### 3.1 Critical (필수 수정, 3건)

#### C-1: proposal §부록 A 헤더 카운트 오류
- **위치**: `uscp-proposal-v2.md` §부록 A 헤더 (라인 339 근처)
- **현상**: 헤더에 "28건 + 사전 협의 3건 + PDF 명시 3건 = 34건"으로 표기
- **실제**: A.1~A.10 합산 = 33행, A.11 = 6행, 총 39행
- **원인**: 1차 미팅 28건의 핵심 결정이 본문에서 33개 세부 항목으로 세분화됨
- **수정안**: 헤더를 "33건 + 사전 협의 3건 + PDF 명시 3건 = 39건"으로 정정

#### C-2: RFI §부록 B.3 헤더 카운트 오류
- **위치**: `uscp-rfi-response-v2.md` §부록 B.3 헤더 (라인 735 근처)
- **현상**: 헤더에 "(28건)"으로 표기
- **실제**: 본문 #1~#33 (33행)
- **수정안**: 헤더를 "(33건)"으로 정정

#### C-3: AUTH-12 시민 회원가입 신규 항목 누락
- **위치**: `uscp-rfi-response-v2.md` §부록 A.1 회원가입·인증 표
- **현상**: AUTH-01~AUTH-08만 8개 행 기재
- **실제**: Design §3.2·§7.3은 "AUTH-12 (시민 회원가입 신규) — 1차 미팅 신규"로 명시
- **수정안**: AUTH-09~AUTH-12 신규 항목 추가 (특히 AUTH-12에 "1차 미팅 신규" 라벨)

### 3.2 Major (개선 권장, 2건)

#### M-1: RFI §9.3 AI 챗봇 가산 범위 산식 오류
- **위치**: `uscp-rfi-response-v2.md` §9.3 견적 영향 변수 (라인 398)
- **현상**: "AI 챗봇·자동 수집 도입 시: +600~1,800만 원"
- **estimate §5와 비교**: AI 챗봇 도입 +600~1,000만 + AI 자동 수집 +800~1,200만 = +1,400~2,200만
- **수정안**: RFI 표기를 estimate와 일치시키거나 두 항목 분리 명시
  - 옵션 a: "AI 챗봇 도입 시: +600~1,000만 / AI 자동 수집 도입 시: +800~1,200만"
  - 옵션 b: "AI 챗봇 + 자동 수집 도입 시: +1,400~2,200만"

#### M-2: 부록 A 해결된 항목 취소선 + ✅ 표기 누락
- **위치**: 세 산출물의 부록 A 전체
- **현상**: Design §7.3 "해결·제거된 항목은 ~~취소선~~ + ✅로 표기" 규칙 위반
- **실제**: 1차 미팅으로 해결·제거된 항목(SUB·VOTE·CMT·NOTIF·GATE·MNT·CMS·PERM 일부)이 부록 A에서 단순 누락된 것처럼 표기
- **수정안**: 발주처에 "의도적 미적용"임을 명확히 전달하기 위해 해결된 기능 ID를 ~~취소선~~ + ✅ 표기 추가
- **우선순위**: 작성 시간 여유 시 적용 (필수 아님)

### 3.3 Minor (참고, 2건)

#### m-1: estimate §2.2 산식 체인 직접 산술 vs 라운딩 표시값
- **위치**: `estimate-v2.md` §2.2 산식
- **현상**: 산식 체인 직접 산술은 4,400~9,300이지만 표시값 5,000~8,500
- **실제**: §3.3 단계별 합계(4,200~6,900 + 800~1,600)가 정확한 도출 경로
- **수정안**: §2.2 산식 끝에 "(단계별 합계 기준 라운딩 후 값)" 단서 추가

#### m-2: RFI 부록 C 유사 수행 실적 플레이스홀더 미기재
- **위치**: `uscp-rfi-response-v2.md` §1.1·부록 C
- **현상**: 회사명·대표자·실적 모두 `[당사명]`·`[프로젝트명]` 플레이스홀더
- **수정안**: 발주처 제출 전 실제 데이터로 채워야 함 (현 시점에는 정상)

## 7. 결론

### 7.1 평가

V2.0 산출물 3종은 **사업 본질·범위·견적·핵심 결정 사항 추적성**에서 우수한 수준(99%, gap-detector 평가)을 보였으나, **표면 표기 정확성**(카운트·신규 항목 표기·산식 일치)에서 5건의 보완 사항이 발견(88%, design-validator 평가)되었습니다.

### 7.2 즉시 조치 사항

Critical 3건은 명백한 표면 오류이므로 즉시 수정 가능:
1. proposal 부록 A 헤더 카운트 정정
2. RFI 부록 B.3 헤더 카운트 정정
3. RFI 부록 A에 AUTH-12 추가

### 7.3 발주처 제출 전 추가 필요 사항 (현 PDCA 사이클 외)

- RFI §1.1 회사 정보 실제 데이터 입력
- RFI 부록 C 유사 수행 실적 포트폴리오 입력
