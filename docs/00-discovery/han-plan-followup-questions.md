## -1. 1차 미팅 결정 사항 (2026-05-11)

### -1.1 운영 모델 본질 변경 — 비로그인 제보 → 로그인 회원 제보

발주처와의 1차 미팅에서 한혜진 계획안의 핵심 운영 모델(바텀업 시민 자유 제보 + 게이트키핑)을 **로그인 회원 제보 모델**로 변경하기로 결정되었습니다. 이는 한혜진 계획안의 가장 큰 본질적 변경 사항이며, 다수의 협의 항목이 자동 해결되는 효과가 있습니다.

**변경 전 (한혜진 계획안 v2.0)**:
```
1단계: 비로그인 시민 자유 제보 (로그인 없이 가능)
   ↓ 게이트키핑
2~6단계 진행
```

**변경 후 (1차 미팅 결정)**:
```
0단계: 회원가입 (개인정보처리방침·이용약관 동의 필수, 만 14세 이상)
   ↓
1단계: 로그인 사용자 제보
   ↓ 게이트키핑
2~6단계 진행
```

### -1.2 영역별 결정 사항 요약

#### 인증·회원 제도

- **회원가입 필수**: 제보자도 사전 회원가입 필요
- **개인정보처리방침·이용약관 동의**: 회원가입 시 필수 절차
- **만 14세 이상**: 약관에 명시하여 우회 (법정대리인 동의 시스템 미구현)
- **광고성 정보 발송 계획 없음**: 정보통신망법 별도 동의 절차 불필요
- **양교 SSO·학사 시스템 연동 없음**: 공주대·충남대 학생·교수도 별도 회원가입

#### 알림

- **알림 채널: 이메일 단일**
- **알림 발송 시점: 모든 단계 변경 시 발송 (단계별 알림)**
- **운영자 알림 라우팅**: 미결정 (2차 협의)

#### CMS·콘텐츠

- **소개 페이지(USCP 소개·5개 지역 안내·참여주체 안내)**: 정적 HTML 제공
- **공지사항·이벤트 공지**: WYSIWYG 에디터 제공
- **약관(개인정보처리방침·이용약관)**: WYSIWYG 에디터 + 버전 관리
- **메인 배너**: 미결정 (CMS 미도입 방침이라 고정 배너 추정, 2차 협의)

#### 멘토단·학생팀

- **멘토 풀**: 가입된 사용자 중 관리자가 선정 (사전 등록제 변형)
- **학생팀 구성**: 관리자가 가입된 사용자 중 직접 선정·구성 (관리자 주도형)
- **매칭 방식**: 운영자 수동 매칭
- **멘토 수락·거절 절차**: 미결정 (2차 협의)

### 0.1  계획안의 알림 관련 명시 (전수 조사)

| 위치 | 명시 내용 |

### 0.2 발생하는 모순 (비로그인 제보 ↔ 알림)

```
[ 계획안 운영 모델]
1단계: 비로그인 시민 제보 (로그인 없이 가능)
   ↓
2단계: 검토중 → 알림 발송???
   ↓
[질문] 로그인하지 않은 시민에게 어떻게 알림을 보내는가?
```

**가능한 답변 시나리오** (모두 별도 정책·구현 필요):

| 시나리오 | 동작 방식 | 추가 요건 |
|---------|---------|----------|
| **A. 이메일 알림** | 제보 시 이메일 주소 입력 필수 | 이메일 발송 인프라(SMTP) + 개인정보 수집 동의 + 발송 스케줄러 |
| **B. SMS 알림** | 제보 시 휴대폰 번호 입력 필수 | SMS API 계약·요금(건당 8~15원) + 개인정보 수집 동의 + 본인인증 검토 |
| **C. 카카오 알림톡** | 비즈니스 채널 + 휴대폰 번호 | 카카오 비즈니스 계약 + 템플릿 사전 승인 + 비용 |
| **D. 추적 토큰 + 회신 페이지** | 제보 시 추적 코드 발급 → 시민이 직접 조회 | 알림 발송 인프라 불필요 / 사용자 경험 약함 |
| **E. 이메일 + 토큰 혼합** | 이메일 입력 선택 — 입력하면 푸시, 안 하면 토큰 | 가장 유연하나 구현 복잡 |

**비용·난이도 비교**:
| 시나리오 | 운영 비용(연) | 사용자 경험 | 추천도 |
|---------|-----------:|:----------:|:----:|
| A. 이메일 | 거의 무료 (SMTP 기본 포함) | 중 | ⭐⭐⭐⭐ |
| B. SMS  | 100~500만 원 (건당 과금) | 상 | ⭐⭐ |
| C. 알림톡 | 50~200만 원 (건당 7~10원) | 상 | ⭐⭐⭐ |
| D. 토큰 | 0원 | 하 | ⭐⭐ |
| E. 혼합 | 거의 무료~중간 | 상 | ⭐⭐⭐⭐⭐ |

→ **권장**: **시나리오 A (이메일 단일)** — 비용·구현·사용자 경험 균형. 향후 SMS·알림톡 확장 가능 구조로 설계.

## 7-A. CMS 도입 여부 의사결정 (콘텐츠 업데이트 주기 기반)

### 7-A.1 배경

 계획안에 **"CMS"라는 명시적 표현·기능은 없으며**, USCP 소개 페이지(플랫폼 소개·5개 지역 안내·참여주체 안내)·메인 배너·약관(개인정보처리방침·이용약관)의 편집 방법이 명시되지 않았음. CMS 도입 여부는 **콘텐츠 업데이트 주기·편집 주체·법적 의무**의 함수.

### 7-A.2 콘텐츠별 업데이트 특성 (당사 추정)

| 콘텐츠 | 예상 업데이트 빈도 | 편집 주체 (가정) | CMS 필요도 |
|--------|:----------------:|:--------------:|:--------:|
| 플랫폼 소개 | 사업 단계 변경 시 (연 0~1회) | 실무자 작성 → 개발사 반영? | 🟢 낮음 |
| 5개 지역 안내 | 지역별 사업·협력기관 변경 시 (연 2~4회) | 실무자 직접? | 🟡 중간 |
| 참여주체 안내 | 정책 변경 시 (연 0~1회) | 실무자 작성 → 개발사 반영? | 🟢 낮음 |
| 메인 배너 | 이벤트·공지·시즌 (월 1회 이상 가능) | 실무자 직접 (대부분) | 🔴 높음 |
| 개인정보처리방침 | 법령·운영 변경 시 (불규칙) | 실무자 + 법무 검토 | 🔴 **법적 의무** |
| 이용약관 | 운영 정책 변경 시 (불규칙) | 실무자 + 법무 검토 | 🔴 **법적 의무** |

### 7-A.3 의사결정 트리

```
질문 1: 약관(개인정보처리방침·이용약관) 변경 시 즉시 반영해야 하는가?
   YES (법적 의무) → CMS 필수 (최소 약관 편집기)
   NO              → 다음 질문

질문 2: 배너 운영을 사업 운영 중에 변경할 계획이 있는가?
   YES → CMS 권장 (배너 관리 모듈)
   NO  → 다음 질문

질문 3: 5개 지역 안내·참여주체 안내가 사업 진행 중 변경될 가능성이 있는가?
   YES → 단순 CMS (페이지 편집기)
   NO  → 정적 HTML (CMS 불필요)
```

### 7-A.4 옵션별 견적

| 옵션 | 포함 범위 | 추가 비용 | 적합 시나리오 |
|------|----------|---------:|-------------|
| **옵션 0: CMS 미도입** | 모두 정적 HTML | 0원 | 콘텐츠 거의 변경 없음 + 약관 변경 시 매번 개발사 호출 가능 |
| **옵션 1: 최소 CMS** | 약관 편집기만 | +200~400만 원 | 법적 의무 해소 + 그 외는 정적 |
| **옵션 2: 표준 CMS** ✅ | 약관 + 배너 + 단순 페이지 편집기 | +400~600만 원 | 일반적 운영 가능 (당사 권장) |
| **옵션 3: 풍부한 CMS** | 옵션 2 + 스마트 에디터(WYSIWYG·동영상·SNS 임베드·지도) | +700~1,000만 원 | 풍부한 콘텐츠 표현 필요 시 |

### 7-A.7 옵션 0 선택 시 발주처가 감수해야 할 리스크

1. 약관 변경 시 매번 개발사 유지보수 호출 → **변경 1회당 약 50~100만 원 유상 유지보수 비용** 발생 가능
2. 법령 변경(개인정보보호법 개정 등)에 신속 대응 어려움 → 개발사 일정에 따라 며칠~수 주 지연
3. 5개 지역의 리빙랩 운영 현황을 소개 페이지에 반영하려면 정적 HTML 매번 수정 필요

### 7-A.8 당사 권고

> 5개 지역 안내·배너의 운영 변경 의향이 있다면 **옵션 2(표준 CMS, +400~600만 원)** 가 가장 균형 있음.
> 발주처 회신 전 당사 입장은 **옵션 2를 기본 견적에 포함**, 답변에 따라 0/1/3로 조정.
