## 1. 사업 개요

### 1.1 사업 목적

1차년도에 구축된 USCP 플랫폼을 고도화합니다. 1차년도에 정적 페이지로 제공되던 **소개 페이지를 관리자가 직접 편집할 수 있는 콘텐츠 관리(CMS) 체계**를 신규 구축하고, 관리자 운영 편의성을 높이는 **자동화·시각화·알림 기능**, **소셜 로그인**, **AI 챗봇 서비스**를 추가하여 플랫폼의 활용도를 극대화합니다.

### 1.2 사업 범위

2차년도 고도화는 크게 두 파트로 구성됩니다.

- **파트 A. 기능 고도화**: 1차년도 플랫폼의 사용성·편의성 강화 (자동화, 시각화, 알림, 소셜 로그인 등)
- **파트 B. AI 챗봇**: 플랫폼 데이터 기반 AI 질의응답 서비스 신규 구축

## 2. 파트 A. 기능 고도화

1차년도에 구축된 핵심 기능 위에 운영 편의성과 사용자 경험을 강화하는 기능을 추가합니다. 1차년도에서 "기능은 되지만 수동으로 관리"했던 부분을 **자동화하고 시각화**합니다.

### 2.1 소셜 로그인 추가

1차년도의 이메일 자체 로그인에 소셜 로그인을 추가하여 사용자의 가입/로그인 허들을 낮춥니다.

| 기능 | 설명 |
|------|------|
| 카카오 로그인 | 카카오 계정으로 간편 로그인. 별도 회원가입 없이 카카오 인증으로 자동 회원 생성 |
| 네이버 로그인 | 네이버 계정으로 간편 로그인 |
| 구글 로그인 | 구글 계정으로 간편 로그인 |
| 계정 연동 | 기존 이메일 계정에 소셜 계정을 연결하여 어떤 방식으로든 로그인 가능 |

### 2.2 프로젝트 관리 고도화

1차년도의 3단계 관리 위에 5단계 프로세스 옵션과 부가 기능을 추가합니다. 모든 프로젝트에 일괄 적용하는 것이 아니라, **프로젝트별로 관리자가 운영 수준을 선택**할 수 있도록 설계합니다.

#### 프로세스 선택 기능

프로젝트의 관리 권한을 가진 참여자(프로젝트 관리자)가 프로세스 모드를 선택합니다.

| 모드 | 단계 | 설명 | 활성화되는 기능 |
|------|------|------|--------------|
| **기본 (3단계)** | 발주 → 진행중 → 완료 | 1차년도와 동일. 산출물 업로드 중심의 간소한 관리 | 산출물 업로드/다운로드 |
| **상세 (5단계)** | 탐색 → 실행 → 개발 → 검증 → 활용 | 리빙랩 5단계 프로세스에 따라 단계별로 관리 | 단계별 산출물 분류, 마일스톤, 팀별 현황, 칸반 보드 |

프로젝트팀이 상세 관리 기능을 사용하지 않는 경우 관련 탭과 메뉴가 노출되지 않아, 불필요한 복잡성 없이 1차년도와 동일한 방식으로 운영할 수 있습니다.

#### 상세 모드(5단계) 선택 시 활성화되는 기능

| 기능 | 설명 |
|------|------|
| 5단계 프로세스 | 탐색 → 실행 → 개발 → 검증 → 활용 단계 전환. 프로젝트 상세에 5단계 시각화 표시 |
| 단계별 산출물 분류 | 산출물 업로드 시 해당 단계를 선택하여 단계별로 결과물을 분류·조회 |
| 마일스톤 관리 | 프로젝트별 마일스톤(중간 목표)을 등록합니다. 제목, 담당자, 기한, 진행 상태를 관리합니다. |
| 마일스톤 기한 알림 | 마일스톤 기한 D-7, D-3에 관리자에게 자동 알림 발송 |
| 팀별 산출물 현황 | 프로젝트 상세에서 팀별로 산출물 제출 현황을 한눈에 조회 |

#### 프로젝트별 권한 관리

프로젝트 참여자에게 역할별 권한을 부여하여, 시스템 관리자가 아닌 **프로젝트 참여자가 직접 해당 프로젝트를 관리**할 수 있도록 합니다.

| 프로젝트 역할 | 권한 | 설명 |
|-------------|------|------|
| **프로젝트 관리자** | 프로젝트 설정/수정, 모드 선택, 팀 구성, 마일스톤 관리, 상태 전환 | 해당 프로젝트의 운영 전반을 담당하는 참여자 (교수 등) |
| **팀장** | 팀 멤버 관리, 산출물 등록/삭제, 게시판 글 작성 | 소속 팀의 결과물 제출과 멤버 관리 |
| **팀원** | 산출물 등록, 게시판 글 작성, 조회 | 참여 활동과 결과물 업로드 |
| **열람자** | 공개 정보 조회만 가능 | 프로젝트에 배정되지 않은 일반 사용자 |

- 시스템 관리자(센터 실무자)는 모든 프로젝트에 대해 전체 권한을 가짐
- 프로젝트 관리자는 시스템 관리자가 지정하거나, 프로젝트 생성 시 자동 배정

#### 공통 고도화 기능 (모드 무관)

| 기능 | 설명 |
|------|------|
| 칸반 보드 | 관리자 전용 화면에서 전체 프로젝트를 단계 열로 시각화. 3단계/5단계 프로젝트가 함께 표시됨 |
| 산출물 등록 알림 | 참여자가 새 산출물을 업로드하면 프로젝트 관리자에게 자동 알림 |
| 이슈→프로젝트 전환 | 이슈 상세 페이지에서 원클릭으로 해당 이슈를 기반으로 프로젝트를 자동 생성. 생성 시 모드 선택 |
| 사업화 연계 이력 | 텍스트 메모 → 연계 사업명, 기관, 연계 시기, 상태를 구조화된 데이터로 관리 |

## 3. 파트 B. AI 챗봇 서비스

플랫폼에 축적된 데이터를 기반으로 사용자의 질문에 자동 응답하는 AI 챗봇 서비스를 구축합니다.

### 3.1 챗봇 인터페이스

| 기능 | 설명 |
|------|------|
| 챗봇 위젯 | 모든 페이지 하단 우측에 상시 노출되는 채팅 버블. 클릭 시 대화창이 열립니다. |
| 대화 화면 | 사용자가 질문을 입력하면 AI가 플랫폼 데이터를 기반으로 답변합니다. 말풍선 형태의 대화 UI. |
| 빠른 질문 버튼 | 자주 묻는 질문을 버튼으로 제공합니다. "리빙랩이란?", "참여 방법은?", "진행 중인 프로젝트는?" 등 |
| 대화 이력 | 로그인 사용자의 이전 대화 내용이 유지됩니다. |
| 전체 화면 모드 | 모바일에서 전체 화면 채팅을 지원합니다. |

### 3.2 AI 엔진

| 기능 | 설명 |
|------|------|
| LLM 연동 | 국가대표 AI 또는 상용 AI API(Claude/GPT)를 연동합니다. 발주처와 협의하여 선정합니다. |
| RAG 파이프라인 | 플랫폼의 소개 페이지, 프로젝트 정보, 성공사례, FAQ 등을 벡터 DB에 인덱싱하고, 사용자 질문 시 관련 문서를 검색하여 AI에 컨텍스트로 전달합니다. 이를 통해 플랫폼 데이터에 기반한 정확한 답변을 생성합니다. |
| 출처 표시 | AI 답변에 참고한 문서의 출처(페이지 링크)를 함께 표시하여 정보의 신뢰성을 확보합니다. |
| 데이터 자동 갱신 | 플랫폼 데이터가 변경(프로젝트 등록, CMS 편집 등)되면 벡터 DB를 자동 업데이트합니다. |

### 3.3 관리자 기능

| 기능 | 설명 |
|------|------|
| 대화 로그 조회 | 사용자들이 챗봇에 질문한 내용과 AI 응답을 관리자가 확인할 수 있습니다. |
| 미응답 질문 관리 | AI가 적절히 답변하지 못한 질문 목록을 확인하여 FAQ 개선에 활용합니다. |
| 데이터 소스 관리 | AI가 참조하는 문서를 추가/제거할 수 있습니다. 외부 PDF 등도 인덱싱에 추가 가능합니다. |

### 3.4 기술 구성

| 항목 | 기술 | 설명 |
|------|------|------|
| LLM | 국가대표 AI / Claude API / GPT API | 발주처 선택. "국가대표 AI와 함께하는 사회공헌 플랫폼" 홍보 효과 |
| 벡터 DB | ChromaDB 또는 Qdrant | 온프레미스 배포 가능. Docker 컨테이너로 구동 |

## 11. 추가 제안: AI 기반 산출물 분석 및 성과 평가 자동화

본 견적 범위에는 포함되지 않으나, AI 챗봇 인프라(RAG + LLM)가 구축된 이후 추가 도입을 제안합니다.

### 11.1 개요

프로젝트 팀이 산출물을 업로드하면 AI가 자동으로 분석하고, 프로젝트 관리자가 최종 승인하는 워크플로우입니다. 관리자는 AI가 정리해준 분석 결과만 확인하고 판단하면 되므로, **운영 부담을 대폭 줄이면서 체계적인 성과 관리**가 가능합니다.

### 11.2 처리 흐름

```
팀원/팀장이 산출물 업로드
    ↓
AI가 산출물 자동 분석
  - 문서 완성도 평가 (목차/분량/필수 항목 충족 여부)
  - 팀별 제출 현황 집계 (제출/미제출/지연)
  - 프로젝트 전체 진행률 산출
  - 이전 단계 산출물 대비 연속성 체크
    ↓
분석 결과를 프로젝트 관리자에게 알림
  - "A팀 탐색 단계 보고서 제출됨. 완성도 85%, 필수 항목 충족"
  - "전체 5팀 중 3팀 제출 완료, 2팀 미제출 (기한 D-3)"
    ↓
프로젝트 관리자가 분석 결과 확인 후
  - 승인 → 단계 완료 처리 / 다음 단계 전환
  - 보완 요청 → 팀에게 피드백 전달
  - 반려 → 사유 작성 후 재제출 요청
```

### 11.3 기능 상세

| 기능 | 설명 |
|------|------|
| AI 산출물 분석 | 업로드된 산출물(문서, 보고서)을 AI가 자동 분석하여 완성도, 필수 항목 충족 여부, 핵심 내용 요약을 생성합니다. |
| 팀별 제출 현황 자동 집계 | 프로젝트 내 전체 팀의 산출물 제출 상태(제출/미제출/지연)를 자동 집계하여 대시보드에 표시합니다. |
| AI 성과 평가 리포트 | 프로젝트 단계 완료 시 AI가 해당 단계의 전체 산출물을 종합 분석하여 성과 평가 초안을 생성합니다. 프로젝트 관리자가 검토 후 확정합니다. |
| 승인 워크플로우 | AI 분석 결과를 기반으로 프로젝트 관리자가 산출물을 승인/보완요청/반려 처리합니다. 처리 결과는 해당 팀에게 자동 알림됩니다. |
| 미제출 자동 알림 | 산출물 제출 기한이 임박하거나 초과된 팀에게 자동으로 독촉 알림을 발송합니다. |

### 11.4 도입 조건

- 2차년도 파트 B(AI 챗봇)의 RAG 파이프라인 및 LLM 연동이 완료된 상태에서 추가 개발
- 동일한 AI 인프라를 재활용하므로 추가 인프라 비용 없음
- 예상 추가 비용: **1,000~1,500만원** (별도 협의)

### 11.5 기대 효과

| 현재 (수동) | 도입 후 (AI 자동화) |
|-----------|------------------|
| 관리자가 산출물을 하나하나 열어서 확인 | AI가 자동 분석 후 요약 결과만 확인 |
| 팀별 제출 현황을 수동 집계 | 대시보드에 자동 표시 |
| 미제출 팀에 개별 연락 | 기한 D-7, D-3에 자동 알림 |
| 성과 평가 보고서를 처음부터 작성 | AI 초안 생성 → 관리자 검토/확정 |
