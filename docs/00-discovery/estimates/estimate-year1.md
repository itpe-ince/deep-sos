# USCP 1차년도 구축 견적서

> **프로젝트**: 온라인 사회공헌 플랫폼 (USCP, Union Social Contribution Platform)<br/>
> **발주처**: 국립공주대학교 지역사회특화센터<br/>
> **발주 방식**: 수의계약<br/>
> **공급가액**: 49,500,000원(부가세 포함)<br/>
> **구축 기간**: 계약일로부터 5개월<br/>
> **유지보수**: 1년 무상<br/>
> **작성일**: 2026-04-14

---

## 1. 사업 개요

### 1.1 사업 목적

국립공주대학교 2025 글로컬대학 본지정 사업의 핵심 과제인 "온라인 사회공헌 플랫폼(USCP)"을 구축합니다. 대전·공주·예산·세종 4개 캠퍼스의 리빙랩 프로젝트를 체계적으로 관리하고, 지역사회에 성과를 공유하는 대외 소통 채널을 제공합니다.

### 1.2 사업 범위

센터 관리자 1인이 효율적으로 운영할 수 있는 플랫폼을 목표로, 핵심 기능에 집중하여 구축합니다.

---

## 2. 구축 기능

### 2.1 회원·인증 시스템

사용자 가입과 로그인, 역할별 접근 제어를 제공합니다.

| 기능 | 설명 |
|------|------|
| 회원가입 | 이메일 + 비밀번호 기반 자체 회원가입. 가입 시 이름, 소속, 역할을 선택합니다. |
| 로그인 | 이메일과 비밀번호로 로그인합니다. 로그인 상태는 세션 기반으로 유지됩니다. |
| 역할 관리 | 관리자, 교수, 학생, 시민 4종의 역할을 구분합니다. 역할에 따라 접근 가능한 메뉴와 기능이 달라집니다. |
| 프로필 관리 | 사용자가 자신의 기본 정보(이름, 소속, 연락처 등)를 수정할 수 있습니다. |
| 비밀번호 찾기 | 가입한 이메일로 비밀번호 재설정 링크를 발송합니다. |

### 2.2 리빙랩 프로젝트 관리

플랫폼의 핵심 기능입니다. 관리자가 리빙랩 프로젝트를 등록하고, 참여 팀과 산출물을 관리합니다.

| 기능 | 설명 |
|------|------|
| 프로젝트 등록/수정 | 관리자가 프로젝트를 생성합니다. 제목, 설명, 수행 기간, 예산, 캠퍼스, SDGs 연계 목표 등의 정보를 입력합니다. |
| 상태 관리 | 프로젝트의 진행 상태를 **발주 → 진행중 → 완료** 3단계로 관리합니다. 관리자가 수동으로 상태를 전환합니다. |
| 프로젝트 종료 | 참여 팀이 최종 산출물을 업로드하면 프로젝트가 완료 처리됩니다. |
| 팀 구성 | 프로젝트에 참여하는 팀을 생성하고, 팀장을 지정하고, 멤버를 배정합니다. 3개 참여 그룹(학생, 지역시민, 교수) 각각에 팀이 구성됩니다. |
| 산출물 관리 | 참여자가 결과물(보고서, 데이터, 프로토타입 등)을 파일로 업로드합니다. 관리자도 산출물을 등록할 수 있습니다. 업로드된 파일은 다운로드할 수 있습니다. |
| 문서 양식 | 관리자가 보고서 양식, 활동 기록 양식 등의 템플릿을 등록하면, 참여자가 다운로드받아 작성 후 결과물로 다시 업로드합니다. 산출물 관리 기능에 "양식 템플릿" 유형으로 통합됩니다. |
| 사업화 연계 메모 | 프로젝트가 외부 사업(창업, 정책 반영 등)으로 연결된 경우, 프로젝트 상세 페이지에 연계 정보를 텍스트로 기록합니다. |
| 공개 프로젝트 목록 | 비로그인 사용자도 볼 수 있는 프로젝트 목록 페이지입니다. 카드 형태의 그리드로 표시되며, 상태별 필터가 제공됩니다. |
| 공개 프로젝트 상세 | 프로젝트의 상세 정보를 탭으로 구분하여 표시합니다. 산출물, 참여 멤버, 연결된 기관 정보를 확인할 수 있습니다. |
| 관리자 프로젝트 관리 | 관리자 전용 화면에서 전체 프로젝트를 테이블 형태로 조회하고, 상태를 변경하고, 멤버를 관리합니다. |

### 2.3 지역 문제 관리

지역사회의 문제를 플랫폼에 등록하고, 리빙랩 프로젝트와 연결합니다.

| 기능 | 설명 |
|------|------|
| 이슈 등록 | 센터 관리자가 지역 문제를 등록합니다. 제목, 상세 설명, 카테고리(환경/안전/교통/복지/문화/기타), 캠퍼스, 현장 사진을 입력합니다. 시민이 직접 등록하는 것이 아니라 관리자가 조사 후 등록하는 탑다운 방식입니다. |
| 이슈 목록 | 비로그인 사용자도 볼 수 있는 이슈 목록 페이지입니다. 카드 리스트 형태로 표시되며, 카테고리별 필터가 제공됩니다. |
| 이슈 상세 | 이슈의 상세 정보를 표시합니다. 문제 설명, 현장 사진, 연결된 프로젝트 정보를 확인할 수 있습니다. |
| 프로젝트 연결 | 이슈 상세 페이지에서 관련 리빙랩 프로젝트를 링크합니다. |
| 관리자 이슈 관리 | 관리자 전용 화면에서 전체 이슈를 테이블 형태로 조회하고, 상태(등록/검토중/해결중/해결완료/보류)를 변경합니다. |

### 2.4 참여 기관 / MOU 관리

리빙랩 프로젝트에 참여하는 외부 기관과 협약(MOU) 정보를 관리합니다.

| 기능 | 설명 |
|------|------|
| 기관 등록/수정/삭제 | 관리자가 참여 기관의 정보를 등록합니다. 기관명, 유형(지자체/기업/연구기관/NGO/대학), 담당자명, 연락처, 주소, 웹사이트를 입력합니다. |
| MOU 관리 | 기관별로 체결된 MOU(협약)를 등록합니다. 협약 제목, 시작일, 종료일, 상태(협의중/체결/만료/해지), 비고를 관리합니다. |
| 프로젝트 연결 | 기관을 특정 프로젝트에 연결하여, 해당 프로젝트에 어떤 기관이 참여하고 있는지 확인할 수 있습니다. |

### 2.5 정적 소개 페이지

본 사업은 **비용 최적화 및 핵심 기능 위주**로 구축됩니다. 플랫폼의 대외 홍보용 소개 페이지는 비용 절감을 위해 정적(하드코딩) 방식으로 제공되며, 최초 1회 발주처의 콘텐츠를 반영하여 개발합니다. 관리자가 직접 편집하는 기능(CMS)은 추후 적용 검토가 필요합니다.

| 기능 | 설명 |
|------|------|
| 소개 페이지 | 플랫폼 소개(about), 참여 방법 안내(guide), 캠퍼스별 리빙랩, FAQ 등을 정적 페이지로 제공합니다. 최초 콘텐츠는 발주처 제공 원고를 기반으로 개발 단계에 반영합니다. |
| 성공사례 안내 페이지 | 성공사례 영역은 공지사항/자료실로 안내 링크를 제공합니다. 전용 관리 기능은 추후 검토가 필요합니다. |
| 메인 배너 | 메인 홈 배너는 정적 이미지 2~3종을 개발 단계에 적용합니다. 이후 배너 변경은 유지보수 범위 내 단순 이미지 교체로 처리합니다. |

> **참고**: 소개 페이지 WYSIWYG 편집기, 성공사례 전용 관리, 배너 CRUD 등 콘텐츠 관리(CMS) 기능 및 자동화·알림 기능은 비용 절감을 위해 본 사업 범위에서 제외되었으며, 추후 적용 검토가 필요합니다. 본 사업 기간 동안 정적 콘텐츠 변경이 필요한 경우 개발사의 유지보수 지원을 통해 처리합니다.

### 2.6 커뮤니티

플랫폼 이용자 간 소통과 정보 공유를 위한 게시판 기능을 제공합니다.

| 기능 | 설명 |
|------|------|
| 공지사항 | 관리자가 등록하는 공식 공지 게시판입니다. 상단 고정, 중요 표시 기능을 지원합니다. |
| 이벤트 공지 | 리빙랩 행사, 설명회, 워크숍 등 이벤트 일정을 안내합니다. 일시, 장소, 참여 대상, 첨부 파일을 등록합니다. |
| 자유 게시판 | 로그인 사용자가 자유롭게 글을 작성할 수 있는 게시판입니다. 댓글 기능을 지원합니다. |
| 자료실 | 리빙랩 관련 자료(가이드, 양식, 보고서 등)를 파일로 등록하고 다운로드할 수 있는 게시판입니다. 카테고리별 분류를 지원합니다. |
| 프로젝트별 게시판 | 각 리빙랩 프로젝트에 종속된 게시판입니다. 프로젝트 참여자가 프로젝트 관련 논의, 자료 공유, 활동 기록을 작성합니다. 프로젝트 상세 페이지의 탭으로 접근합니다. |

### 2.7 메인 홈 / 관리자 대시보드

| 기능 | 설명 |
|------|------|
| 메인 홈 페이지 | 플랫폼의 첫 화면입니다. 히어로 배너, 주요 지표(프로젝트 수/이슈 수/참여 기관 수 등), 최근 등록된 프로젝트와 이슈, 4개 캠퍼스 소개가 표시됩니다. |
| 관리자 대시보드 | 관리자 로그인 후 표시되는 전체 현황 화면입니다. 프로젝트 수, 이슈 수, 기관 수, 최근 등록된 항목이 텍스트 수치로 표시됩니다. |
