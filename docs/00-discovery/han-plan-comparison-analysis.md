## 🎯 발주처 최종 의사결정 (2026-05-08 업데이트)

**담당자 한혜진 연구원 의견 수렴 결과**, 아래 사항이 확정되었으며 본 보고서의 권고를 재정렬합니다.

### A. 관점 전환 — 책임자 시점 → 실무자 시점

기존 인터뷰(04-13)의 운영 원칙은 **사업 책임자 관점**에서 도출된 것으로, "운영 인력 부담 최소화"가 핵심 가치였습니다. 한혜진 연구원의 계획안(04-29)은 **실무자 관점**에서 작성되어 "지역문제·리빙랩 관리 기능 강화"가 핵심 가치입니다.

| 관점 영역 | 기존 (책임자 시점) | 변경 (실무자 시점) | 설계 영향 |
|----------|------------------|------------------|----------|
| **본질 질문** | "어떻게 적은 인력으로 운영할 것인가?" | "어떻게 일을 잘 할 수 있는 도구를 만들 것인가?" | 설계 철학 전환 |
| **자동화 가치** | 운영 부담 경감의 수단 | 부차적 — 실무자는 **충분한 관리 도구**가 더 중요 | AI 자동화 우선순위 하락 |
| **관리 기능 단순화** | 1인이 다룰 수 있을 만큼 간소 | **풍부한 관리 기능 + 명확한 워크플로우** 우선 | 기능 강화 방향 |
| **시민 참여 모델** | 탑다운 (운영 부담 통제) | **바텀업 (시민 제보 + 게이트키핑)** | 운영 부담 수용 |
| **운영 인력** | 가정에 포함 (1인) | **가정 자체를 두지 않음** — 발주처가 운영 인력은 결정 | 설계 가정에서 제외 |

> **핵심 결론**: 본 사업의 설계 원칙에서 **운영 인력 가정을 명시하지 않음**. 1인이든 다수든 운영 인력 규모는 발주처의 결정 영역이며, 시스템은 **사용자(시민·실무자·멘토·학생) 관점에서 필요한 기능을 충실히 제공**하는 것이 우선.

| **AI 산출물 분석** | 산출물 검수 부하 | **실무자가 직접 검수·관리하는 것이 본질**. AI 보조 불필요 | ✅ 영향 없음 |
| **AI 프로젝트 지원** | 운영자 의사결정 지원 | **실무자가 직접 결정·기록하는 것이 본질**. AI 브리핑 불필요 | ✅ 영향 없음 |

> **결론**: 5개 모듈 제거가 **한혜진 계획안의 핵심 기능에 어떤 부정적 영향도 주지 않음**. 안전하게 제거 가능.

### F. 실무자 관점 전환에 따른 설계 원칙 재정립

운영 인력 가정(1인/다수)은 **본 사업 설계 가정에서 제외**합니다. 시스템은 **사용자(시민·실무자·멘토·학생) 관점에서 필요한 기능을 충실히 제공**하는 데 집중합니다.

| 영역 | 기존 (책임자 관점) | 신규 (실무자 관점) |
|------|------------------|------------------|
| 핵심 가치 | 운영 부담 최소화 | **관리 기능 충실성** |
| 권한 분리 | 자동화로 권한 단순화 | **실무자별 책임 분리·게이트키핑 통제** |
| 자동화 | AI로 부담 경감 우선 | **실무자가 직접 일을 잘 하는 도구** 우선 |
| 알림 설계 | 알림 최소화·필터링 | **담당자·지역·단계별 알림 라우팅** (실무 효율) |
| 게이트키핑 | 부담 줄이기 | **명확한 워크플로우·이력·이의제기 절차** |
| 콘텐츠 운영 | 정적 페이지로 단순화 | **실무자가 직접 편집·운영하는 도구** |
| 매뉴얼 | 단일 가이드 | **역할별·지역별 실무 매뉴얼** |

> **핵심**: 한혜진 계획안의 본질은 **"실무자가 일을 잘 할 수 있게 도와주는 도구"** 의 구현이며, "운영 인력을 줄이기 위한 자동화"가 아닙니다. 따라서:
> - AI 자동화 모듈은 본질에 부합하지 않아 제외 → 발주처 결정과 일치
> - 권한·감사·개인정보는 **운영 인력 규모와 무관하게** 법·실무 차원에서 필수 → 협의 완료
> - CMS·시스템 설정은 **실무자가 직접 운영**하기 위한 도구 → 강력 권고 유지
