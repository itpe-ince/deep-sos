> **Version**: 1.0
> **Date**: 2026-04-08
> **Status**: Draft
> **적용 대상**: SOS랩(USCP) 전체 프론트엔드

## 1. 기본 원칙

### 1.1 핵심 원칙

| 원칙 | 설명 | 적용 기준 |
|------|------|----------|
| **일관성** | 동일 패턴은 동일 UI로 구현 | 컴포넌트 재사용률 80% 이상 |
| **최소 인터랙션** | 사용자 목적 달성까지 최소 클릭 | 핵심 태스크 3클릭 이내 |
| **맥락 유지** | 현재 작업 흐름을 끊지 않는 UI | 모달 최소화, 페이지 이동 최소화 |
| **예측 가능성** | 사용자가 결과를 예측 가능 | 동작 전 피드백, 일관된 위치 |
| **접근성** | 다양한 환경에서 사용 가능 | WCAG 2.1 AA 준수 |

### 1.2 사용자 유형별 고려사항

| 사용자 | 기술 수준 | UI 고려 |
|--------|----------|--------|
| 학생 | 높음 (모바일 네이티브) | 모바일 퍼스트, 빠른 인터랙션 |
| 교수 | 중간 | 데스크톱 우선, 명확한 네비게이션 |
| 시민 | 다양 (낮을 수 있음) | 직관적, 큰 터치 타겟, 쉬운 언어 |
| 지자체 공무원 | 중간 | 대시보드 중심, 데이터 가독성 |
| 관리자 | 높음 | 효율성 중심, 단축키 지원 |

## 2. 레이아웃 체계

### 2.1 페이지 구조

```
┌──────────────────────────────────────────────────────────┐
│  GNB (Global Navigation Bar) — 고정 (sticky top)          │
│  높이: 56px (Desktop) / 48px (Mobile)                     │
├──────────┬───────────────────────────────────────────────┤
│          │  Breadcrumb (해당 시)                          │
│  LNB     ├───────────────────────────────────────────────┤
│ (Left    │                                               │
│  Nav)    │  Page Content                                 │
│          │  ┌───────────────────────────────────────┐    │
│ 너비:     │  │  Content Area                         │    │
│ 240px    │  │  (스크롤 가능 영역)                     │    │
│          │  │                                       │    │
│ 접힘:     │  │                                       │    │
│ 64px     │  │                                       │    │
│ (아이콘)  │  └───────────────────────────────────────┘    │
│          │                                               │
│          │  Pagination / Load More (해당 시)              │
├──────────┴───────────────────────────────────────────────┤
│  Footer (공개 페이지만) — 비로그인 페이지에만 표시            │
└──────────────────────────────────────────────────────────┘
```

### 2.2 반응형 브레이크포인트

| 이름 | 범위 | 레이아웃 변화 |
|------|------|------------|
| **Mobile** | 0 ~ 767px | LNB 숨김 → 하단 탭 바로 전환, 1 컬럼 |
| **Tablet** | 768px ~ 1023px | LNB 접힘(64px 아이콘), 1~2 컬럼 |
| **Desktop** | 1024px ~ 1439px | LNB 펼침(240px), 2~3 컬럼 |
| **Wide** | 1440px+ | 최대 너비 1440px 중앙 정렬, 3 컬럼 |

### 2.3 GNB (Global Navigation Bar)

```
Desktop:
┌──────────────────────────────────────────────────────────┐
│ [로고]  [캠퍼스: 대전 ▾]           [검색] [알림🔔] [프로필]│
└──────────────────────────────────────────────────────────┘

Mobile:
┌──────────────────────────────────────────────────────────┐
│ [≡]  [로고]                              [검색] [알림🔔] │
└──────────────────────────────────────────────────────────┘
```

**고정 규칙:**
- GNB는 항상 **상단 고정 (sticky)**
- 스크롤 시에도 유지
- z-index: 1000

### 2.4 LNB (Left Navigation Bar)

```
Desktop (펼침 240px):           Mobile (하단 탭 바):
┌────────────────────┐         ┌────────────────────────┐
│ 📋 홈               │         │ 홈 | 이슈 | 랩 | 공헌 | 더보기│
│ 🔍 이슈             │         └────────────────────────┘
│   └ 전체 이슈        │         높이: 56px, 하단 고정
│   └ 내 이슈          │
│   └ 지도 보기        │
│ 🧪 리빙랩            │
│   └ 프로젝트         │
│   └ 내 프로젝트      │
│ 🤝 사회공헌          │
│   └ 봉사활동         │
│   └ 내 포트폴리오    │
│ 💬 Q&A              │
│ 🌱 ESG              │
│ ⚙️ 관리 (admin)     │
└────────────────────┘
```

**고정 규칙:**
- Desktop: 좌측 고정, 페이지 콘텐츠와 독립 스크롤
- Mobile: 하단 탭 바로 전환, **하단 고정 (fixed bottom)**
- z-index: 900

### 2.5 콘텐츠 영역 그리드

| 타입 | 설명 | 사용처 |
|------|------|--------|
| **1 컬럼** | 전체 너비 사용 | 상세 페이지, 폼, 에디터 |
| **2 컬럼 (7:3)** | 메인 + 사이드 | 상세 + 메타정보 |
| **2 컬럼 (5:5)** | 균등 분할 | 비교 뷰 |
| **3 컬럼 카드** | 카드 그리드 | 목록 (프로젝트, 활동) |
| **리스트** | 테이블/리스트 뷰 | 이슈 목록, 관리자 |

그리드 거터: 24px (Desktop) / 16px (Mobile)

## 3. 모달 표준

### 3.1 모달 사용 원칙

> **원칙: 모달 사용을 최소화한다.**

| 상황 | 모달 사용 | 대안 |
|------|:---:|------|
| 간단 확인 (삭제, 취소) | ✅ | - |
| 짧은 입력 (댓글, 투표 사유) | ✅ | - |
| 경고/에러 알림 | ✅ | Toast 우선 검토 |
| 긴 폼 (이슈 등록, 프로젝트 생성) | ❌ | **별도 페이지** 또는 **Drawer** |
| 복잡한 선택 (멤버 선택, 필터) | ❌ | **Drawer** 또는 **인라인 확장** |
| 상세 정보 조회 | ❌ | **별도 페이지** |

### 3.2 모달 구조 규격

```
┌────────────────────────────────────────┐ ← 상단 고정
│  [제목]                         [✕ 닫기]│    (sticky)
├────────────────────────────────────────┤
│                                        │
│  컨텐츠 영역                            │ ← 내부 스크롤
│  (overflow-y: auto)                    │    (이 영역만 스크롤)
│                                        │
│  · 컨텐츠가 길어도 헤더/푸터 유지         │
│  · max-height: calc(100vh - 200px)     │
│                                        │
├────────────────────────────────────────┤
│  [취소]                    [확인/저장]   │ ← 하단 고정
└────────────────────────────────────────┘    (sticky)
```

### 3.3 모달 크기 규격

| 타입 | 너비 | 용도 |
|------|------|------|
| **Small** | 400px (max 90vw) | 확인/삭제, 간단 알림 |
| **Medium** | 560px (max 90vw) | 짧은 폼, 선택 |
| **Large** | 720px (max 90vw) | 복잡한 폼 (가급적 Drawer 대체) |

### 3.4 모달 행동 규칙

| 규칙 | 설명 |
|------|------|
| **Backdrop 클릭 닫기** | 폼 입력 없는 경우: 닫기 허용. 폼 입력 중: 닫기 방지 또는 확인 |
| **ESC 키 닫기** | 항상 허용 |
| **중첩 모달 금지** | 모달 위에 모달 절대 불가. 필요 시 현재 모달 내용 교체 |
| **스크롤 잠금** | 모달 열림 시 배경 body 스크롤 잠금 |
| **포커스 트랩** | 모달 내부에 포커스 가둠 (접근성) |
| **애니메이션** | 열기: fade-in + scale(0.95→1), 150ms. 닫기: fade-out, 100ms |

### 3.5 Drawer (사이드 패널)

모달 대신 복잡한 인터랙션에 사용:

```
Desktop (오른쪽 Drawer):
┌──────────────────────┬──────────────┐
│                      │  Drawer      │ ← 너비: 480px
│  기존 페이지 유지      │  ┌─────────┐│    (max 50vw)
│  (딤처리, 클릭 닫기)   │  │ Header  ││
│                      │  ├─────────┤│
│                      │  │ Content ││ ← 내부 스크롤
│                      │  │ (scroll)││
│                      │  ├─────────┤│
│                      │  │ Footer  ││
│                      │  └─────────┘│
└──────────────────────┴──────────────┘

Mobile (하단 Sheet):
┌──────────────────────────────────────┐
│  기존 페이지 (딤처리)                   │
├──────────────────────────────────────┤
│  ── 핸들 ──                           │ ← 드래그로 높이 조절
│  Sheet Header                        │    min: 40vh
│  ─────────────────────────────────   │    max: 90vh
│  Sheet Content (scroll)              │
│  ─────────────────────────────────   │
│  Sheet Footer (Actions)              │
└──────────────────────────────────────┘
```

**Drawer 사용 케이스:**
- 필터 패널 (이슈 필터, 프로젝트 필터)
- 멤버 선택/관리
- 이슈 빠른 보기 (목록에서 클릭 시)
- 알림 목록 상세

## 4. 네비게이션 및 고객 여정

### 4.1 여정 최소화 원칙

> **원칙: 핵심 태스크는 3클릭 이내로 도달한다.**

| 태스크 | 최대 클릭 | 경로 |
|--------|:---:|------|
| 이슈 등록 | 2 | 홈 → [이슈 등록] 버튼 → 폼 페이지 |
| 이슈 공감 | 1 | 이슈 카드에서 바로 공감 버튼 |
| 프로젝트 조회 | 2 | LNB → 프로젝트 목록 → 상세 |
| 봉사활동 신청 | 2 | 봉사 목록 → 상세 → [신청] 버튼 |
| Q&A 질문 | 2 | Q&A → [질문하기] → 폼 페이지 |
| 알림 확인 | 1 | GNB 알림 아이콘 → 드롭다운 |

### 4.2 페이지 전환 규칙

| 규칙 | 설명 |
|------|------|
| **SPA 네비게이션** | Next.js Link로 클라이언트 사이드 전환 (깜빡임 없음) |
| **뒤로 가기 보장** | 브라우저 뒤로 가기 시 이전 상태 복원 (스크롤 위치 포함) |
| **URL 상태 반영** | 필터, 정렬, 페이지 등 주요 상태를 URL 쿼리에 반영 |
| **로딩 표시** | 페이지 전환 시 상단 progress bar (NProgress 스타일) |
| **에러 복구** | 페이지 로드 실패 시 재시도 버튼 + 이전 페이지 유지 |

### 4.3 Breadcrumb 규칙

```
홈 > 리빙랩 > 청년정착 리빙랩 프로젝트 > 프로젝트 상세
```

- **3depth 이상**에서만 표시
- 각 단계 클릭 시 해당 페이지로 이동
- Mobile에서는 "← 이전" 버튼으로 대체

## 5. 스크롤 체계

### 5.1 스크롤 원칙

| 영역 | 스크롤 방식 |
|------|-----------|
| **GNB** | 스크롤 안 됨 (sticky top) |
| **LNB** | 독립 스크롤 (메뉴 항목이 많을 때) |
| **Page Content** | 메인 스크롤 영역 |
| **모달 Content** | 내부 스크롤 (Header/Footer 고정) |
| **Drawer Content** | 내부 스크롤 (Header/Footer 고정) |
| **테이블** | 가로 스크롤 허용 (모바일), 세로는 페이지 스크롤 |

### 5.2 무한 스크롤 vs 페이지네이션

| 사용처 | 방식 | 근거 |
|--------|------|------|
| 이슈 목록 | **페이지네이션** | URL로 상태 공유, 특정 위치 북마크 |
| 프로젝트 목록 | **페이지네이션** | 같은 이유 |
| 댓글/답변 | **더보기 버튼** | 양이 제한적, 전체 맥락 유지 |
| 알림 목록 | **무한 스크롤** | 최신순 탐색, 북마크 불필요 |
| 타임라인 | **더보기 버튼** | 순서 중요, 맥락 유지 |

### 5.3 Sticky 요소 z-index 체계

| 요소 | z-index | 비고 |
|------|:------:|------|
| 모달 Backdrop | 1100 | |
| 모달 Content | 1200 | |
| Drawer | 1050 | |
| Toast/Snackbar | 1300 | 최상위 |
| GNB | 1000 | |
| LNB | 900 | |
| Sticky 테이블 헤더 | 100 | |
| FAB (Floating Action) | 800 | |
| Dropdown/Popover | 500 | |

## 6. 인터랙션 패턴

### 6.1 폼 표준

#### 폼 레이아웃
```
┌──────────────────────────────────────────┐
│  페이지 제목                              │
│  설명 텍스트 (선택)                        │
├──────────────────────────────────────────┤
│                                          │
│  라벨 *                                  │
│  ┌──────────────────────────────────┐   │
│  │ Input                            │   │
│  └──────────────────────────────────┘   │
│  도움말 텍스트 또는 에러 메시지             │
│                                          │
│  라벨                                    │
│  ┌──────────────────────────────────┐   │
│  │ Textarea                         │   │
│  │                                  │   │
│  └──────────────────────────────────┘   │
│                                          │
│  라벨                                    │
│  ┌──────────────────────────────────┐   │
│  │ Select ▾                         │   │
│  └──────────────────────────────────┘   │
│                                          │
├──────────────────────────────────────────┤
│              [취소]  [저장] ← 우측 정렬    │
│  * 필수 입력                              │
└──────────────────────────────────────────┘
```

#### 폼 규칙

| 규칙 | 설명 |
|------|------|
| **필수 표시** | 라벨 뒤 `*` (빨간색) |
| **실시간 검증** | blur 시 개별 필드 검증 (submit 전에도) |
| **에러 표시** | 필드 아래 빨간 텍스트, 필드 border 빨간색 |
| **저장 버튼 위치** | 폼 하단 우측 (취소 왼쪽, 저장 오른쪽) |
| **긴 폼 분할** | 5개 이상 필드 시 섹션 분리 또는 스텝 폼 |
| **이탈 방지** | 입력 중 페이지 이탈 시 확인 다이얼로그 |
| **자동 저장** | 긴 폼(리빙랩 프로젝트 등)에서 30초 간격 임시 저장 |

### 6.2 피드백 패턴

| 유형 | UI | 지속 시간 | 사용처 |
|------|------|:------:|--------|
| **Toast (성공)** | 하단 중앙, 초록색 | 3초 자동 닫힘 | 저장 완료, 투표 완료 |
| **Toast (정보)** | 하단 중앙, 파란색 | 3초 자동 닫힘 | 상태 변경 알림 |
| **Toast (경고)** | 하단 중앙, 주황색 | 5초 자동 닫힘 | 주의 필요 사항 |
| **Toast (에러)** | 하단 중앙, 빨간색 | 수동 닫기 | API 에러, 검증 실패 |
| **인라인 에러** | 필드 아래 | 수정 시까지 | 폼 검증 에러 |
| **빈 상태** | 콘텐츠 영역 중앙 | - | 데이터 없음, 검색 결과 없음 |
| **스켈레톤** | 콘텐츠 형태 유지 | 로딩 완료까지 | 초기 로딩 |
| **Spinner** | 버튼 내부 | 완료까지 | 버튼 클릭 후 처리 중 |

### 6.3 확인/삭제 패턴

```
삭제 확인 모달 (Small):
┌──────────────────────────────────┐
│  이슈를 삭제하시겠습니까?          │
│                                  │
│  삭제된 이슈는 복구할 수 없습니다.  │
│                                  │
│         [취소]  [삭제] ← 빨간색   │
└──────────────────────────────────┘

· 파괴적 액션(삭제): 버튼 빨간색, "삭제" 명시
· 비파괴적 액션(취소, 닫기): 기본 버튼
· 되돌릴 수 없는 액션: 대상명 재입력 요구 (관리자 기능)
```

## 7. 컴포넌트 표준

### 7.1 버튼

| 타입 | 용도 | 스타일 |
|------|------|--------|
| **Primary** | 주요 액션 (저장, 등록, 확인) | 채움 배경, 흰 텍스트 |
| **Secondary** | 보조 액션 (취소, 뒤로) | 테두리, 배경 투명 |
| **Danger** | 파괴 액션 (삭제) | 빨간 채움 또는 빨간 텍스트 |
| **Ghost** | 부가 액션 (더보기, 링크) | 텍스트만, 호버 시 배경 |
| **Icon** | 아이콘 액션 (공감, 북마크) | 아이콘만, 호버 시 배경 |
| **FAB** | 주요 생성 액션 (이슈 등록) | 원형, 우하단 고정 (Mobile) |

**버튼 크기:**
| 크기 | 높이 | 패딩 | 폰트 | 사용처 |
|------|:----:|:----:|:----:|--------|
| sm | 32px | 8px 12px | 13px | 테이블 내, 보조 |
| md | 40px | 10px 16px | 14px | 일반 (기본값) |
| lg | 48px | 12px 24px | 16px | 폼 제출, CTA |

**버튼 상태:**
- Default → Hover (배경 어둡게) → Active (더 어둡게) → Disabled (50% 투명도)
- Loading: 텍스트 → Spinner 교체, 클릭 비활성화

### 7.2 카드

```
┌──────────────────────────────────┐
│  [카테고리 뱃지]         [상태 뱃지]│
│                                  │
│  제목 (최대 2줄, 말줄임)           │
│  설명 텍스트 (최대 3줄, 말줄임)     │
│                                  │
│  ┌────────┐                      │
│  │ 이미지  │ (선택, 있을 때만 표시) │
│  └────────┘                      │
│                                  │
│  👍 12  💬 5  👁 234    2시간 전  │
└──────────────────────────────────┘
```

**카드 규칙:**
- 호버 시: 그림자 강화 (elevation 변경)
- 클릭: 전체 카드가 클릭 영역 (내부 버튼 제외)
- 텍스트 말줄임: 제목 2줄, 설명 3줄 max
- 반응형: Desktop 3열 → Tablet 2열 → Mobile 1열

### 7.3 뱃지/태그

| 용도 | 스타일 | 예시 |
|------|--------|------|
| **상태** | 채움 배경, 라운드 | `접수됨` `처리중` `완료` |
| **카테고리** | 테두리, 투명 배경 | `학사` `시설` `행정` |
| **캠퍼스** | 색상 구분 | `대전` `공주` `예산` `세종` |
| **SDGs** | 공식 색상 사용 | `SDG 4` `SDG 11` |
| **역할** | 아이콘 + 텍스트 | `교수` `학생` `시민` |

### 7.4 테이블

```
┌──────┬──────────────┬────────┬────────┬────────┐
│ 선택 │ 제목 ▾ (정렬) │ 상태   │ 담당자  │ 일자    │ ← 헤더 sticky
├──────┼──────────────┼────────┼────────┼────────┤
│ □    │ 이슈 제목 1   │ 접수됨 │ 홍길동  │ 04/08  │
│ □    │ 이슈 제목 2   │ 처리중 │ 김교수  │ 04/07  │
├──────┴──────────────┴────────┴────────┴────────┤
│  ◀ 1 2 3 4 5 ▶        20개씩 보기 ▾            │ ← 하단 고정
└────────────────────────────────────────────────┘
```

**테이블 규칙:**
- 헤더: **sticky** (콘텐츠 스크롤 시 상단 고정)
- 페이지네이션: 테이블 하단 고정
- 모바일: 가로 스크롤 허용 또는 카드 뷰로 전환
- 정렬: 컬럼 헤더 클릭으로 정렬 전환 (▲▼ 표시)
- 행 호버: 배경색 변경
- 빈 상태: "등록된 데이터가 없습니다" + CTA 버튼

## 11. 접근성 표준

### 11.1 필수 준수 사항

| 항목 | 기준 | 적용 |
|------|------|------|
| **색상 대비** | 4.5:1 이상 (본문), 3:1 이상 (큰 텍스트) | 모든 텍스트 |
| **키보드 네비게이션** | Tab으로 모든 인터랙티브 요소 접근 | 전 페이지 |
| **포커스 표시** | 포커스 링 (2px solid, Primary 색상) | 인터랙티브 요소 |
| **Alt 텍스트** | 모든 의미 있는 이미지에 alt 필수 | 이미지 |
| **ARIA 라벨** | 아이콘 버튼에 aria-label 필수 | 아이콘 버튼 |
| **터치 타겟** | 최소 44x44px | 모바일 |
| **애니메이션** | prefers-reduced-motion 존중 | 모든 애니메이션 |

### 11.2 스크린 리더 고려

| 요소 | 처리 |
|------|------|
| 상태 변경 알림 | `aria-live="polite"` |
| 모달 열림 | `role="dialog"`, `aria-modal="true"` |
| 로딩 | `aria-busy="true"`, 완료 시 알림 |
| 에러 메시지 | `role="alert"` |
| 페이지 전환 | 제목 요소에 포커스 이동 |

### 12.1 로딩 전략

|------|----------|
