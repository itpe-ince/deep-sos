## 1. 프로토타입 범위 결정

### 1.1 고객 답변 핵심 요약

| 항목 | 결정 사항 |
|------|---------|
| **플랫폼 유형** | ⑤ 종합 플랫폼 (소통 + 시민참여 + 프로젝트관리 + 성과관리) |
| **예상 비용** | 1.05~1.58억 |
| **핵심 사용자** | 교수·학생 + 시민 |
| **인증** | **독자 로그인 체계** (학교 SSO 미연동) |
| **서버 환경** | 대학 전산실 온프레미스 |
| **1차 완료** | 2026년 12월 |
| **운영** | 지역사회특화센터 전담 |
| **사용자 규모** | 약 470명 + 시민 (소규모) |

### 1.2 기능 우선순위 (고객 지정)

| 순위 | 기능 영역 | 비고 |
|:---:|---------|------|
| **1** | 소개 페이지 (CMS) | 단체·제도·참여 방법 안내 |
| **2** | 지역 문제 등록·투표 | BF-1 |
| **3** | 리빙랩 프로젝트 관리 (5단계) | BF-3 (전체 관리) |
| **4** | 봉사활동 관리 + 봉사 인증 | BF-5 + VMS/1365 자동 |
| **5** | ESG/SDGs 성과 관리 | BF-6 |
| **6** | 사업화 연계 | BF-4 |
| **7** | 성과 공개·홍보 | BF-7 |
| **8** | 참여 기관 관리 | BF-2 |
| **9** | 관리자 현황판·성과지표 | 공통 |

### 1.3 프로토타입 검증 목표

프로토타입의 목적은 **코드 구현이 아니라 고객 합의 확보**입니다.
아래 5가지를 확인하는 것이 핵심입니다:

1. **플랫폼 정체성 일치**: 고객이 머리에 그리는 시스템과 일치하는가?
2. **핵심 화면 흐름**: 사용자가 목적을 3클릭 이내로 달성하는가?
3. **기능 우선순위 확인**: 1차 오픈(2026.12) 범위가 현실적인가?
4. **UI 표준 검증**: 모달 최소화, 여정 최소화 원칙이 적절한가?
5. **디자인 방향**: 폰트, 색상, 레이아웃이 발주처 정서에 맞는가?

## 2. 프로토타입 화면 구조

### 2.1 Information Architecture (IA)

```
SOS랩 플랫폼
│
├─ 공개 영역 (비로그인 열람 가능)
│  ├─ 홈 — 플랫폼 소개, 주요 지표, 최신 소식
│  ├─ 플랫폼 소개
│  │  ├─ SOS랩이란?
│  │  ├─ 운영 기관 소개 (지역사회특화센터/ESG센터/국제협력센터)
│  │  ├─ 캠퍼스별 리빙랩 (대전/공주/예산/세종)
│  │  └─ 참여 방법 안내
│  ├─ 지역 문제 — 현황 열람
│  ├─ 리빙랩 프로젝트 — 진행 현황 열람
│  ├─ 성공 사례 — 사례 아카이브
│  ├─ 봉사활동 — 모집 현황 열람
│  ├─ SDGs 현황 — 대시보드
│  ├─ 공지·뉴스
│  └─ 로그인 / 회원가입
│
├─ 사용자 영역 (로그인 필수)
│  ├─ 대시보드 — 내 활동, 알림
│  ├─ 지역 문제 등록·관리
│  ├─ 리빙랩 프로젝트 참여
│  ├─ 봉사활동 신청·내역
│  ├─ 사회공헌 포트폴리오
│  └─ 마이페이지
│
└─ 관리자 영역 (관리자 전용)
   ├─ 통합 대시보드 (KPI, 통계)
   ├─ 콘텐츠 관리 (CMS)
   │  └─ 소개 페이지, 공지, 배너, 메뉴
   ├─ 이슈 관리 (담당 배정, 상태 변경)
   ├─ 프로젝트 관리 (5단계, 멤버, KPI)
   ├─ 봉사활동 관리 (VMS 동기화)
   ├─ 사용자/권한 관리
   └─ 참여 기관·MOU 관리
```

## 3. 핵심 사용자 여정 (User Flows)

### Flow 1: 시민이 지역 문제를 등록하는 흐름

```
[비로그인] 홈 방문
   │
   ↓
[P-01 홈] "지역 문제 제보" 버튼 클릭
   │
   ↓
[P-10 로그인/회원가입] 회원가입
   │
   ↓
[P-12 지역 문제 등록]
   ├─ 카테고리 선택
   ├─ 제목·설명 입력
   ├─ 위치 선택 (지도)
   ├─ 사진 업로드
   └─ 등록 완료
   │
   ↓
[P-06 이슈 상세] 등록된 이슈 확인
   │
   ↓ (알림)
[이메일/푸시] 상태 변경 알림 수신
   │
   ↓
[P-06 이슈 상세] 처리 결과 확인
```

### Flow 2: 관리자가 이슈를 프로젝트로 전환하는 흐름

```
[P-19 관리자 대시보드] 새 이슈 알림 확인
   │
   ↓
[P-22 이슈 관리 목록] 처리 대기 이슈 선택
   │
   ↓
[P-23 이슈 처리 상세]
   ├─ 내용 검토
   ├─ 담당자 배정
   └─ "리빙랩 프로젝트로 전환" 버튼
   │
   ↓
[P-25 프로젝트 관리 상세]
   ├─ 프로젝트 기본 정보 입력
   ├─ 탐색 단계로 자동 설정
   ├─ 멤버 추가
   └─ 첫 마일스톤 등록
   │
   ↓ (자동)
[시민 알림] "제보하신 문제가 프로젝트로 전환되었습니다"
```

### Flow 3: 교수가 리빙랩 프로젝트를 진행하는 흐름

```
[P-11 사용자 대시보드] 내 프로젝트 목록
   │
   ↓
[P-13 프로젝트 참여 상세] 현재 단계 확인
   │
   ↓
   ├─ [P-15 아이디어 보드] 팀원과 아이디어 공동 작성
   │  └─ 카드 추가 → 투표 → 상위 아이디어 채택
   │
   ├─ [P-14 현장 피드백 등록] 현장 방문 후 피드백 등록
   │  └─ 설문 or 체크리스트 or 사진+메모
   │
   └─ 마일스톤 완료 처리
   │
   ↓
[관리자 승인 후]
   │
   ↓
[P-13 프로젝트 참여 상세] 다음 단계로 전환
```

### Flow 4: 관리자가 소개 페이지를 편집하는 흐름

```
[P-19 관리자 대시보드]
   │
   ↓
[P-20 CMS — 소개 페이지 편집]
   ├─ 페이지 목록
   ├─ "SOS랩이란?" 선택
   ├─ WYSIWYG 에디터로 편집
   ├─ 이미지 삽입
   └─ 미리보기 → 발행
   │
   ↓
[공개 영역 P-02] 변경 내용 즉시 반영
```

## 4. 화면별 상세 기획 (핵심 5개)

### P-01. 홈

**목적**: 플랫폼 정체성 전달 + 주요 활동 노출 + 참여 유도

**구성**:
```
┌─────────────────────────────────────────┐
│ GNB (로고 / 메뉴 / 캠퍼스 선택 / 로그인)   │
├─────────────────────────────────────────┤
│ 히어로 배너                               │
│ "대학과 지역이 함께 만드는 사회공헌 플랫폼"  │
│ [지역 문제 제보하기] [리빙랩 둘러보기]     │
├─────────────────────────────────────────┤
│ 주요 지표 (KPI 박스 4개)                  │
│ [해결된 문제 N건] [진행 프로젝트 N건]     │
│ [봉사 시간 N시간] [참여 시민 N명]         │
├─────────────────────────────────────────┤
│ 최근 지역 문제 (카드 3개) — 더보기 →       │
├─────────────────────────────────────────┤
│ 진행 중인 리빙랩 프로젝트 (카드 3개)       │
├─────────────────────────────────────────┤
│ 성공 사례 (카드 3개)                      │
├─────────────────────────────────────────┤
│ 캠퍼스별 리빙랩 (4개 카드 — 대전/공주/예산/세종) │
├─────────────────────────────────────────┤
│ 공지사항 / 이벤트                         │
├─────────────────────────────────────────┤
│ Footer (기관 정보, 연락처, SDGs 아이콘)    │
└─────────────────────────────────────────┘
```
