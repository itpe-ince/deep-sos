## BF-1. 지역 문제 수집·정의·우선순위화

> **목적**: 지역의 문제를 누구나 쉽게 등록하고, 시민의 공감으로 중요도를 결정한다

### 화면: 이슈 목록 (`/issues`)

| # | 기능 | 상세 | 사용자 |
|---|------|------|--------|
| 1-1 | 이슈 목록 조회 | 카드/리스트 뷰 전환. 카테고리·상태·캠퍼스 필터. 최신순/공감순 정렬. 페이지네이션 | 전체 (비로그인 열람 가능) |
| 1-2 | 지도 뷰 | 지도 위에 이슈 핀 표시. 클러스터링. 핀 클릭 시 간략 정보 팝업 → 상세 이동 | 전체 |
| 1-3 | 카테고리별 통계 | 카테고리별 이슈 수, 해결률. 캠퍼스별 비교 차트 | 전체 |
| 1-4 | 검색 | 제목/내용 키워드 검색 | 전체 |

### 화면: 이슈 등록 (`/issues/new`)

| # | 기능 | 상세 | 사용자 |
|---|------|------|--------|
| 1-5 | 이슈 등록 폼 | 제목(필수), 설명(필수), 카테고리 선택(필수), 위치(지도 클릭 또는 GPS), 사진(최대 5장, 10MB/장), 익명 여부 체크 | 로그인 사용자 |
| 1-6 | 위치 선택 | 지도에서 핀 드래그로 위치 지정. GPS 자동 감지 버튼. 주소 자동 변환 | 로그인 사용자 |
| 1-7 | 임시 저장 | 작성 중 자동 임시저장 (30초 간격). 미제출 임시 이슈 목록 관리 | 로그인 사용자 |

### 화면: 이슈 상세 (`/issues/:id`)

| # | 기능 | 상세 | 사용자 |
|---|------|------|--------|
| 1-8 | 이슈 상세 조회 | 제목, 설명, 사진, 위치(지도), 카테고리, 등록일, 조회수 | 전체 |
| 1-9 | 공감 투표 | 공감 버튼 (토글). 현재 공감 수 표시. 중복 불가 | 로그인 사용자 |
| 1-10 | 처리 상태 타임라인 | 접수 → 검토중 → 배정됨 → 처리중 → 완료. 각 단계별 담당자·일시·메모 표시 | 전체 |
| 1-11 | 댓글 | 댓글 작성/수정/삭제. 익명 댓글 옵션. 공식 답변 표시(담당자) | 로그인 사용자 |
| 1-12 | 이슈→프로젝트 연결 | 이 이슈를 해결하기 위한 리빙랩 프로젝트가 생성되면 링크 표시 | 전체 |
| 1-13 | 상태 변경 | 담당자/관리자가 상태를 변경. 변경 시 메모 입력. 알림 발송 | 담당자/관리자 |
| 1-14 | 담당자 배정 | 이슈를 담당자에게 배정. 배정 시 알림 | 관리자 |

| 3-2 | 캠퍼스 특화 뷰 | 캠퍼스별 리빙랩 유형(청년정착/문화재생/고령돌봄/모빌리티) 구분 표시 | 전체 |

| # | 기능 | 상세 | 사용자 |
|---|------|------|--------|
| 3-3 | 프로젝트 개요 | 제목, 설명, 캠퍼스, 기간, 예산, 연계 SDGs, 메이커 단계 | 전체 |
| 3-4 | **5단계 프로세스 표시기** | 탐색→실행→개발→검증→활용. 현재 단계 강조. 각 단계 클릭 시 해당 마일스톤 표시 | 전체 |

| 3-7 | 참여기관 | 협력 기관 목록. 기관 추가 (BF-2 연동). 기관별 역할 표시 | 리더/관리자 |

| 3-9 | 산출물 관리 | 마일스톤별 산출물 업로드 (보고서, 사진, 발표자료, 데이터). 파일 목록 및 다운로드 | 멤버 |
