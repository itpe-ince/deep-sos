## 1. 시스템 개요 (Overview)

**USCP(온라인 사회공헌 플랫폼, Union Social Contribution Platform)** 는 국립공주대학교가 2025 글로컬대학 본지정 사업의 핵심 인프라로 구축한 통합 사회공헌·리빙랩 운영 플랫폼입니다. 대전·공주·예산·세종 4개 캠퍼스에서 진행되는 리빙랩 프로젝트와 지역사회 문제 해결 활동을 단일 디지털 환경에서 통합 관리하며, 학생·시민·교수·외부 전문가(JA 교원)가 협력하여 지역 현안을 발굴·실행·검증·확산하는 전 과정을 지원합니다.

당사는 본 플랫폼의 기획·설계·구축·고도화를 단일 책임으로 수행하였으며, **현대화 풀스택(Next.js + FastAPI + PostgreSQL)** 과 **AI 자동화 파이프라인(RAG · LLM · 벡터 DB)** 을 결합한 차세대 공공 SI 표준 모델로 완성하였습니다.

---

## 2. 구축 목적 (Why)

본 시스템은 발주처 인터뷰와 글로컬대학 실행계획서 분석을 통해 다음의 본질적 과제 해결을 목표로 설계되었습니다.

| 과제 | 시스템 대응 |
|------|------------|
| **1인 운영의 구조적 한계** — 센터 전담 인력 1명이 4개 캠퍼스·1,000명(2029년 목표) JA 교원·연간 수백 건 산출물 관리 | 수동 업무를 시스템·AI로 이전, 운영자는 의사결정에만 집중 |
| **탑다운 운영 원칙 유지** — 시민 자유 등록이 아닌 센터 담당자 검증 후 등록 | 등록·승인 권한은 관리자에 일원화, AI는 후보 제시만 수행 |
| **산출물 중심 관리** — 프로세스 모니터링이 아닌 결과물 평가 | 단계별 산출물 트리·AI 완성도 평가·자동 승인 워크플로우 |
| **정량 KPI 추적** — 융합과정 이수율·JA 교원 수·교과목 운영 등 실행계획서 지표 | KPI 모듈에서 자동 집계 + 목표 대비 달성률 시각화 |

## 3. 시스템 특징 (How)

### 3.1 3단계 진화 로드맵

USCP는 일회성 구축이 아닌 **3단계 점진적 진화**를 전제로 설계되었습니다.

| 단계 | 포커스 | 핵심 산출 |
|------|--------|----------|
| **1차년도** | 핵심 플랫폼 구축 (운영 기반) | 19개 화면 · 79개 기능 — 프로젝트·이슈·기관/MOU·커뮤니티·5종 역할·감사 로그 |
| **2차년도** | 운영 편의성 + 센터 운영 체계 | +14개 화면 · 59개 기능 — CMS 신규·JA 교원·리빙랩 교과목·KPI·거버넌스·시스템 설정 UI |
| **AI 별도 제안** | 업무 자동화 혁신 | +9개 화면 · 55개 기능 — AI 챗봇·지역문제 자동 수집·산출물 분석·프로젝트 지원 |

**누적 규모**: 42개 화면 / 193개 기능 / 최대 18개월 구축

### 3.2 핵심 기능 특징

- **5종 역할 기반 권한 체계**: 시스템관리자·관리자·교수·학생·시민. 메뉴×역할 매트릭스로 동적 접근 제어
- **리빙랩 프로젝트 라이프사이클 관리**: 3단계(기본) 또는 5단계(탐색→실행→개발→검증→활용) 모드 선택. 팀 구성·산출물·기관 연계 통합 관리
- **CMS 자체 운영 체계**: 스마트 에디터(WYSIWYG, 동영상·SNS·지도 임베드, 발행 예약, 이력 관리)로 발주처가 외부 의존 없이 콘텐츠 운영
- **JA(Joint Appointment) 교원 1,000명 관리**: 위촉 계약·전문 분야·활동 이력·공개 프로필. 학사 시스템 연계 가능
- **AI 기반 4대 자동화** *(별도 제안)*:
  - **AI 프로젝트 지원**: 현황 자동 브리핑·리스크 예측·다음 단계 제안·우수 프로젝트 탐지
  - **AI 챗봇 (RAG)**: 플랫폼 데이터 자동 학습·출처 표기·전 페이지 위젯 상시 노출
  - **AI 지역문제 자동 수집**: 뉴스·민원·지자체 공지 자동 분류·요약·중복 제거 → 관리자 1클릭 등록
  - **AI 산출물 분석**: 완성도 자동 평가·누락 항목 지적·승인/보완/반려 워크플로우·자동 독촉

### 3.3 기술 아키텍처

| 계층 | 채택 기술 | 선정 이유 |
|------|----------|----------|
| 프론트엔드 | Next.js 15 (App Router) + React | SSR/SSG·디자인 시스템 통합·장기 유지보수성 |
| 백엔드 | FastAPI (Python) | 고성능 비동기·OpenAPI 자동 문서화·AI 통합 용이 |
| 데이터베이스 | PostgreSQL | ACID 보장·복잡한 관계형 쿼리·공공기관 친화 |
| 인증 | JWT 자체 + OAuth2 (소셜) | 자율성·확장성 |
| 배포 | Docker Compose · 대학 전산실 온프레미스 | 데이터 주권·이관 용이성 |
| AI | LangChain + ChromaDB/Qdrant + LLM API | RAG 운영 표준·국가대표 AI 또는 상용 API 선택 가능 |

## 4. 사업성 (Market Potential)

### 4.1 직접 사업 가치

- **단일 사업 규모**: 누적 9,500만 원~1억 3,000만 원대 (1차+2차+AI 별도 제안)
- **유지보수 정기 매출 전환 가능**: 무상 1년 종료 후 Basic/Standard/Premium 유상 SLA 옵션
- **장기 파트너십**: 5년 글로컬대학 사업 주기와 연동된 단계적 발주 구조

### 4.2 확장 가능성

| 확장 영역 | 적용 시나리오 |
|----------|-------------|
| **타 글로컬대학** | 전국 30개 글로컬대학 본지정 사업 — 동일 모듈 재사용 + 캠퍼스 파라미터화 |
| **지자체 협력** | 충남·세종 등 지자체 리빙랩·시민참여 플랫폼으로 횡전개 |
| **AI 모듈 단독 SaaS화** | 산출물 분석·지역문제 수집 모듈을 분리 SaaS 상품으로 재구성 |

- **사회혁신·SDGs 연계 수요**: ESG 경영·사회공헌 가시화 요구가 대학·공공기관에 확산

## 5. 가치 제안 (Value Proposition)

### 5.1 발주처(국립공주대학교)에 대한 가치

| 가치 영역 | 구체적 효과 |
|----------|-----------|
| **운영 효율화** | 1인 담당자가 1,000명 규모를 운영 가능. 연간 업무시간 약 1,144h → 265h (-77%) |
| **데이터 자산화** | 1차년도부터 축적된 운영 데이터가 2차·AI 단계의 기반이 되어 매몰 비용 없음 |
| **자체 운영 역량 확보** | 2차 CMS 도입 시 발주처가 외부 의존 없이 콘텐츠·KPI·약관 직접 관리 |
| **데이터 주권** | 온프레미스 운영으로 외부 클라우드 의존 없이 학내 데이터 통제 |
| **표준 모델 선점** | 글로컬대학 사업 모범 사례로 정착, 타 캠퍼스 확산 시 주도권 확보 |

### 5.2 사용자(학생·시민·교수·JA 교원)에 대한 가치

- **단일 진입점**: 리빙랩 참여·산출물 등록·평가가 분산 도구 없이 한 화면에서 완결
- **투명한 성과 추적**: 본인 활동 이력이 사회공헌 포트폴리오로 자동 정리되어 진로·연구 활용 가능
- **AI 도우미 상시 지원**: 모든 페이지 우측 하단에서 즉시 안내·검색 가능 (출처 표기로 신뢰성 확보)

### 5.3 사회·정책적 가치

- **지역문제 가시화**: 4개 캠퍼스 단위 지역 현안과 해결 과정이 공개되어 시민 참여·언론 보도·정책 반영 활성화
- **정책 피드백 루프 구축**: 프로젝트 성과 → 정책 제안 → 반영 여부를 거버넌스 모듈에서 추적하여 대학-지자체 협업 실적 축적
- **사회공헌 정량화**: SDGs 17개 지표·캠퍼스별 활동을 데이터로 측정하여 ESG 보고 자료로 활용

- **연간 업무시간 -77% 절감** — 수동 운영 대비 AI 자동화 도입 효과 (실행계획서 가정 기반 추산)
