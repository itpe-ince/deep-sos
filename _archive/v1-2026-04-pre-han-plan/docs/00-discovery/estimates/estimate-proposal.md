## 1. 사업 개요

### 1.1 제안 배경

1차년도(핵심 기능)와 2차년도(CMS + 센터 운영 체계)를 통해 USCP 플랫폼이 안정적으로 운영되면, 다음 두 가지 방향의 확장이 가능합니다.

- **프로젝트 관리 고도화**: 리빙랩 5단계 프로세스 시각화와 마일스톤 관리로 프로젝트 진행 상황을 체계적으로 관리
- **AI 기반 업무 자동화**: 챗봇 서비스, 지역문제 자동 수집, 산출물 자동 분석·평가로 센터 운영자의 업무 부담을 대폭 감소

본 제안서는 이 두 영역을 **단일 패키지**로 묶어 별도 발주 형태로 제안합니다.

### 1.2 사업 범위

별도 제안은 크게 네 파트로 구성됩니다.

- **파트 A. 프로젝트 관리 고도화 (AI 자동화·지원)** — 5단계 프로세스 구조 위에 AI가 프로젝트 운영자의 판단·작업을 지원
- **파트 B. AI 챗봇 서비스** — 플랫폼 데이터 기반 질의응답
- **파트 C. AI 기반 지역문제 자동 수집** — 외부 뉴스/민원 자동 수집 + 분류
- **파트 D. AI 산출물 분석 자동화** — 업로드 산출물 자동 분석 + 승인 워크플로우

## 2. 파트 A. 프로젝트 관리 고도화 — AI 기반 업무 자동화·지원

파트 A의 핵심은 **AI가 프로젝트 운영자의 판단과 작업을 지원**하는 것입니다. 1차년도의 3단계 관리만으로는 AI가 활용할 데이터 구조가 부족하므로, 5단계 프로세스와 마일스톤을 AI 자동화의 기반으로 포함합니다.

### 2.1 프로세스 구조 (AI 자동화의 기반)

AI가 활용할 수 있도록 프로젝트 운영 구조를 체계화합니다.

| 기능 | 설명 |
|------|------|
| 프로세스 모드 선택 | 프로젝트별로 3단계(기본) / 5단계(상세) 모드 선택 |
| 5단계 프로세스 | 탐색 → 실행 → 개발 → 검증 → 활용 단계 전환, 시각화 |
| 단계별 산출물 분류 | 산출물 업로드 시 해당 단계 선택 — AI가 단계별 진도 판단에 활용 |
| 칸반 보드 | 전체 프로젝트를 단계 열로 시각화, 드래그 전환. 3/5단계 통합 표시 |
| 마일스톤 등록 | 제목, 담당, 기한 등록 — AI가 일정 기반 리스크 예측에 활용 |

### 2.2 AI 기반 프로젝트 업무 지원 (핵심)

프로젝트 운영자가 수동으로 했던 현황 파악, 품질 점검, 판단 작업을 AI가 자동으로 수행·지원합니다.

| 기능 | 설명 | 효과 |
|------|------|------|
| **AI 프로젝트 현황 요약** | 프로젝트 관리자 접속 시 AI가 진행 상황을 자동 요약 제시 | "A팀 마일스톤 2건 지연, B팀 산출물 기한 임박" 한눈에 파악 |
| **AI 산출물 적정성 평가** | 업로드된 산출물을 AI가 자동 분석 — 단계 요구사항 충족 여부, 완성도 점수, 누락 항목 지적 | 일일이 열어볼 필요 없이 분석 결과만 확인 |
| **AI 리스크 예측 알림** | 마일스톤 기한·산출물 제출 패턴 기반 지연 리스크 자동 감지 | D-7/D-3 단순 알림을 넘어서 이상 징후 선제 탐지 |
| **AI 다음 단계 제안** | 현재 단계 산출물 완료 시 다음 단계 전환 여부 및 필요 작업 자동 추천 | 단계 전환 판단 부담 경감 |
| **AI 진행률 자동 계산** | 산출물 제출 현황·마일스톤 진척도 기반 프로젝트 진행률 자동 산출 | 수동 집계 불필요 |
| **AI 유사 사례 추천** | 현재 프로젝트와 유사한 과거 프로젝트·성공사례 자동 검색·제안 | 중복 이슈 방지, 베스트 프랙티스 공유 |
| **AI 팀 활동 요약** | 팀별 활동 이력(산출물, 게시글, 댓글)을 주/월 단위 요약 리포트 자동 생성 | 정기 보고서 작성 부담 경감 |
| **AI 우수 프로젝트 탐지** | 산출물 품질·참여도·성과 기반 우수 프로젝트 자동 선별 → 공유회 추천 | 우수 사례 발굴 자동화 |

## 3. 파트 B. AI 챗봇 서비스

플랫폼에 축적된 데이터(CMS 페이지, 프로젝트, 성공사례, 공지사항, 게시판 등)를 기반으로 사용자의 질문에 자동 응답하는 AI 챗봇 서비스를 구축합니다.

### 3.1 챗봇 프론트엔드

| 기능 | 설명 |
|------|------|
| 챗봇 위젯 | 모든 페이지 하단 우측 채팅 버블. 클릭 시 대화창 |
| 대화 화면 | 사용자 질문 → AI 답변 말풍선 UI. 스트리밍 응답 |
| 빠른 질문 버튼 | 자주 묻는 질문 (리빙랩이란?, 참여 방법 등) |
| 대화 이력 | 로그인 사용자의 이전 대화 유지. 대화별 제목 자동 생성 |
| 전체 화면 모드 | 모바일 전체 화면 채팅 (`/chat` 경로) |
| 출처 표시 | AI 답변에 참고 문서 출처(페이지 링크) 표시 |

### 3.2 RAG 파이프라인 / 데이터 운영

| 기능 | 설명 |
|------|------|
| 문서 수집기 | 플랫폼 DB에서 인덱싱 대상 데이터 추출 (CMS, 프로젝트, 성공사례, FAQ, 공지사항) |
| 문서 전처리 | 청크 단위 분할, 메타데이터(출처 URL, 카테고리, 수정일) 부착 |
| 임베딩 생성 | 청크별 벡터 임베딩 생성 |
| 벡터 DB 저장 | ChromaDB/Qdrant에 upsert 저장 |
| 실시간 이벤트 갱신 | 플랫폼 데이터 변경 시 해당 문서만 즉시 재임베딩 |
| 배치 전체 재인덱싱 | 일 1회(야간) 전체 데이터 재수집·재임베딩 (cron 기반) |
| 임베딩 버전 관리 | 재인덱싱 시 이전 버전 보관, 롤백 가능 |
| 인덱싱 상태 모니터링 | 마지막 동기화 시각, 문서/청크 수, 실패 항목 조회 |
| 외부 문서 업로드 | 관리자가 외부 PDF, HWP 등을 수동 업로드 |
| 데이터 소스 관리 | AI 참조 데이터 소스 목록, 활성화/비활성화 |

### 3.3 LLM 연동 / 품질 관리

| 기능 | 설명 |
|------|------|
| LLM API 연동 | 국가대표 AI 또는 상용 API(Claude/GPT) |
| 시스템 프롬프트 관리 | AI 역할, 응답 스타일, 제한사항 편집 |
| 가드레일 | 부적절한 질문 필터링, 플랫폼 범위 외 질문 안내 |
| 대화 로그 (관리자) | 사용자 질문 및 AI 응답 이력 조회 |
| 미응답 질문 관리 | AI가 답변 못한 질문 목록 → FAQ 개선 활용 |
| 응답 피드백 수집 | 사용자 좋아요/싫어요 평가 |
| API 사용량 모니터링 | 월간 호출 수, 토큰 사용량, 비용 추산 |

### 3.4 기술 구성

| 항목 | 기술 | 비고 |
|------|------|------|
| LLM | 국가대표 AI / Claude API / GPT API | 발주처 선택. 국가대표 AI 사용 시 학교 GPU 서버 필요 |
| 벡터 DB | ChromaDB 또는 Qdrant | 온프레미스 배포 가능 |
| RAG 프레임워크 | LangChain | 문서 분할, 임베딩, 검색, 프롬프트 구성 |

### 3.5 운영비 안내

구축비 외 **월간 운영비**가 발생할 수 있습니다.

| 옵션 | 월 운영비 (예상) | 장점 | 전제조건 |
|------|:--------------:|------|---------|
| 국가대표 AI (온프레미스) | 5~10만원 (전기료) | 홍보 효과, 데이터 외부 미전송 | 학교 GPU 서버 필요 |
| 외부 API (Claude/GPT) | 10~30만원 | 별도 서버 불필요 | API 사용료 발주처 부담 |

## 4. 파트 C. AI 기반 지역문제 자동 수집

실행계획서 p.49 "AI 기반 지역문제 수집"에 대응하는 기능입니다. 외부 뉴스·민원·SNS 등에서 지역 현안을 자동 수집하여 관리자에게 이슈 등록 후보로 제시합니다.

| 기능 | 설명 |
|------|------|
| 외부 데이터 수집기 | 지역 뉴스(네이버/다음 뉴스 API), 민원 플랫폼, 지자체 공지사항 등 크롤링 |
| 키워드 필터링 | 관심 키워드(교통/환경/복지/안전 등) 및 지역명 기반 1차 필터링 |
| LLM 기반 자동 분류 | AI가 수집된 콘텐츠를 카테고리별 자동 분류 + 핵심 내용 요약 |
| 이슈 후보 목록 | 자동 수집된 이슈 후보를 관리자가 검토하는 전용 화면 |
| 이슈 등록 전환 | 후보 항목을 기반으로 원클릭으로 정식 이슈 등록 (원문 링크·요약 자동 연동) |
| 수집 로그 모니터링 | 일별 수집 건수, 분류 정확도, 등록 전환률 |

> 이슈 등록 권한은 여전히 관리자에게 있으며, AI는 **후보 제시**만 수행합니다. 관리자가 검토 후 수동으로 등록합니다.

## 5. 파트 D. AI 산출물 분석 및 성과 평가 자동화

2차년도에 구축된 리빙랩 교과목 운영 체계 위에, **AI가 산출물을 자동 분석하여 담당자에게 최종 승인을 요청**하는 워크플로우를 추가합니다.

### 5.1 처리 흐름

```
팀원/팀장이 산출물 업로드
    ↓
AI가 산출물 자동 분석
  - 문서 완성도 평가 (목차/분량/필수 항목 충족 여부)
  - 팀별 제출 현황 집계 (제출/미제출/지연)
  - 프로젝트 전체 진행률 산출
  - 이전 단계 산출물 대비 연속성 체크
    ↓
분석 결과를 프로젝트 관리자에게 알림
  - "A팀 탐색 단계 보고서 제출됨. 완성도 85%, 필수 항목 충족"
  - "전체 5팀 중 3팀 제출 완료, 2팀 미제출 (기한 D-3)"
    ↓
프로젝트 관리자가 분석 결과 확인 후
  - 승인 → 단계 완료 처리 / 다음 단계 전환
  - 보완 요청 → 팀에게 피드백 전달
  - 반려 → 사유 작성 후 재제출 요청
```

### 5.2 기능 상세

| 기능 | 설명 |
|------|------|
| AI 산출물 분석 | 업로드 문서 자동 분석 — 완성도, 필수 항목 충족 여부, 핵심 내용 요약 |
| 팀별 제출 현황 자동 집계 | 팀별 제출 상태(제출/미제출/지연)를 대시보드에 자동 표시 |
| AI 성과 평가 리포트 | 단계 완료 시 전체 산출물 종합 분석 → 평가 초안 생성. 관리자 검토 후 확정 |
| 승인 워크플로우 | AI 분석 결과 기반 승인/보완요청/반려 처리, 결과는 팀에 자동 알림 |
| 미제출 자동 독촉 | 제출 기한 임박/초과 팀에 자동 독촉 알림 |

### 5.3 기대 효과

| 현재 (수동) | 도입 후 (AI 자동화) |
|-----------|------------------|
| 관리자가 산출물을 하나하나 열어서 확인 | AI가 자동 분석 후 요약 결과만 확인 |
| 팀별 제출 현황을 수동 집계 | 대시보드에 자동 표시 |
| 미제출 팀에 개별 연락 | 기한 D-7, D-3에 자동 알림 |
| 성과 평가 보고서를 처음부터 작성 | AI 초안 생성 → 관리자 검토/확정 |
