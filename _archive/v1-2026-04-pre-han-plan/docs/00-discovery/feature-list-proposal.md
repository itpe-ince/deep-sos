# USCP 별도 제안 기능 목록

> **프로젝트**: 온라인 사회공헌 플랫폼 (USCP) — 별도 제안 패키지<br/>
> **총액**: 3,000만원 VAT 포함<br/>
> **전제조건**: 1차년도 플랫폼 구축 완료 후 2차년도와 동시·순차 진행 가능<br/>
> **작성일**: 2026-04-21

## 1. Information Architecture (별도 제안 추가분)

### 1.1 공개 영역 추가

```
USCP
├── ...기존 메뉴 유지...
└── [신규] AI 챗봇 (전체 화면 /chat)
```

### 1.2 관리자 영역 추가

```
관리자
├── 프로젝트 관리 (AI 기반 업무 자동화·지원)
│   ├── [신규] 5단계 프로세스 + 칸반 보드 (AI 활용 기반)
│   ├── [확장] 마일스톤 관리 (AI 리스크 예측 연동)
│   ├── [신규] AI 프로젝트 현황 요약 (접속 시 자동 브리핑)
│   ├── [신규] AI 기반 산출물 적정성 평가 (단계별 요구 충족 여부)
│   ├── [신규] AI 리스크 예측 알림 (지연·품질 이슈 선제 탐지)
│   ├── [신규] AI 다음 단계 제안 (전환 여부·필요 작업 추천)
│   ├── [신규] AI 진행률 자동 계산 (산출물·마일스톤 기반)
│   ├── [신규] AI 유사 사례 추천 (과거 프로젝트·성공사례)
│   ├── [신규] AI 팀 활동 주/월 요약 리포트
│   └── [신규] AI 우수 프로젝트 자동 탐지 (공유회 추천)
├── [신규] AI 챗봇 관리
│   ├── 대화 로그 조회
│   ├── 미응답 질문 관리
│   ├── 응답 피드백 현황
│   ├── 데이터 소스 관리
│   ├── 인덱싱 상태 모니터링
│   ├── 수동 재인덱싱 실행
│   ├── 외부 문서 업로드 (PDF, HWP)
│   ├── 시스템 프롬프트 편집
│   └── API 사용량 모니터링
├── [신규] 지역문제 자동 수집 관리
│   ├── 이슈 후보 목록 (AI 자동 수집)
│   ├── 후보 검토 및 정식 이슈 등록 전환
│   └── 수집 로그 모니터링 (수집 건수, 정확도)
└── [신규] 산출물 분석 대시보드
    ├── 팀별 제출 현황 자동 집계
    ├── AI 분석 결과 확인
    ├── 승인 / 보완 / 반려 워크플로우
    └── AI 성과 평가 리포트 조회
```

## 2. 기능 목록

## 파트 A. 프로젝트 관리 고도화 — AI 기반 업무 자동화·지원

> 파트 A는 **AI가 프로젝트 운영자의 판단과 작업을 지원**하는 것에 포커스합니다.
> 5단계 프로세스 관리는 AI가 활용할 수 있는 운영 구조의 기반으로서 포함됩니다.

### 2.1 프로세스 구조 (AI 자동화의 기반)

| # | 기능 | 설명 |
|---|------|------|
| P-A01 | 프로세스 모드 선택 | 프로젝트별로 3단계(기본) / 5단계(상세) 모드 선택 |
| P-A02 | 5단계 프로세스 | 탐색 → 실행 → 개발 → 검증 → 활용 단계 전환. 5단계 시각화 |
| P-A03 | 단계별 산출물 분류 | 산출물 업로드 시 해당 단계 선택 — AI가 단계별 진도 판단에 활용 |
| P-A04 | 칸반 보드 | 전체 프로젝트를 단계 열로 시각화, 드래그 앤 드롭 전환. 3/5단계 통합 표시 |
| P-A05 | 마일스톤 등록 | 마일스톤 제목, 담당, 기한 등록 — AI가 일정 기반 알림·리스크 예측에 활용 |

### 2.2 AI 기반 프로젝트 업무 지원 (핵심)

| # | 기능 | 설명 |
|---|------|------|
| P-A06 | **AI 프로젝트 현황 요약** | 프로젝트 관리자 접속 시 AI가 진행 상황을 자동 요약 제시 ("A팀 마일스톤 2건 지연, B팀 산출물 기한 임박 등") |
| P-A07 | **AI 기반 산출물 적정성 평가** | 업로드된 산출물을 AI가 자동 분석 — 단계 요구사항 충족 여부, 완성도 점수, 누락 항목 지적 |
| P-A08 | **AI 리스크 예측 알림** | 마일스톤 기한·산출물 제출 패턴 기반 지연 리스크 자동 감지 및 선제적 알림 (D-7, D-3 이전에도 이상 징후 탐지) |
| P-A09 | **AI 다음 단계 제안** | 현재 단계 산출물 완료 시 AI가 다음 단계 전환 여부 및 필요 작업 자동 추천 |
| P-A10 | **AI 진행률 자동 계산** | 산출물 제출 현황·마일스톤 진척도 기반 프로젝트 진행률 자동 산출 |
| P-A11 | **AI 유사 사례 추천** | 현재 프로젝트와 유사한 과거 프로젝트·성공사례를 AI가 자동 검색·제안 |
| P-A12 | **AI 팀 활동 요약** | 팀별 활동 이력(산출물, 게시글, 댓글 등)을 AI가 주/월 단위 요약 리포트 생성 |
| P-A13 | **AI 기반 우수 프로젝트 탐지** | 산출물 품질·참여도·성과 지표 기반 AI가 우수 프로젝트 자동 선별 → 공유회 추천 |

## 파트 B. AI 챗봇 서비스

### 2.4 챗봇 프론트엔드

| # | 기능 | 설명 |
|---|------|------|
| A-B01 | 챗봇 위젯 | 모든 페이지 하단 우측 채팅 버블 |
| A-B02 | 대화 화면 | 사용자 질문 → AI 답변 말풍선 UI. 스트리밍 응답 |
| A-B03 | 빠른 질문 버튼 | 자주 묻는 질문 버튼 (리빙랩이란?, 참여 방법 등) |
| A-B04 | 대화 이력 | 로그인 사용자의 이전 대화 유지. 대화별 제목 자동 생성 |
| A-B05 | 전체 화면 모드 | 모바일 전체 화면 채팅 (/chat 경로) |
| A-B06 | 출처 표시 | AI 답변에 참고 문서 출처(페이지 링크) 표시 |

### 2.5 RAG 파이프라인 / 데이터 운영

| # | 기능 | 설명 |
|---|------|------|
| A-B07 | 문서 수집기 | 플랫폼 DB에서 인덱싱 대상 데이터 추출 (CMS, 프로젝트, 성공사례, FAQ, 공지사항) |
| A-B08 | 문서 전처리 | 청크 단위 분할, 메타데이터(출처 URL, 카테고리, 수정일) 부착 |
| A-B09 | 임베딩 생성 | 청크별 벡터 임베딩 생성 |
| A-B10 | 벡터 DB 저장 | ChromaDB/Qdrant에 upsert 저장 |
| A-B11 | 실시간 이벤트 갱신 | 플랫폼 데이터 변경 시 해당 문서만 즉시 재임베딩 |
| A-B12 | 배치 전체 재인덱싱 | 일 1회(야간) 전체 데이터 재수집·재임베딩 (cron 기반) |
| A-B13 | 임베딩 버전 관리 | 재인덱싱 시 이전 버전 보관, 롤백 가능 |
| A-B14 | 인덱싱 상태 모니터링 | 마지막 동기화 시각, 문서/청크 수, 실패 항목 조회 |
| A-B15 | 외부 문서 업로드 | 관리자가 외부 PDF, HWP 등을 수동 업로드 |
| A-B16 | 데이터 소스 관리 | AI 참조 데이터 소스 목록, 활성화/비활성화 |

### 2.6 LLM 연동 / 품질 관리

| # | 기능 | 설명 |
|---|------|------|
| A-B17 | LLM API 연동 | 국가대표 AI 또는 상용 API(Claude/GPT) |
| A-B18 | 시스템 프롬프트 관리 | AI 역할, 응답 스타일, 제한사항 편집 |
| A-B19 | 가드레일 | 부적절한 질문 필터링, 플랫폼 범위 외 질문 안내 |
| A-B20 | 대화 로그 (관리자) | 사용자 질문 및 AI 응답 이력 조회 |
| A-B21 | 미응답 질문 관리 | AI가 답변 못한 질문 목록 → FAQ 개선 활용 |
| A-B22 | 응답 피드백 수집 | 사용자 좋아요/싫어요 평가 |
| A-B23 | API 사용량 모니터링 | 월간 호출 수, 토큰 사용량, 비용 추산 |

## 파트 D. AI 산출물 분석 자동화

> 2차년도 리빙랩 교과목 운영 체계 위에 AI 자동 분석·승인 워크플로우를 추가합니다.

### 2.10 AI 산출물 분석

| # | 기능 | 설명 |
|---|------|------|
| A-D01 | AI 산출물 분석 | 업로드 문서 자동 분석 — 완성도, 필수 항목 충족 여부, 핵심 내용 요약 |
| A-D02 | 팀별 제출 현황 자동 집계 | 팀별 제출 상태(제출/미제출/지연) 대시보드 자동 표시 |
| A-D03 | AI 성과 평가 리포트 | 단계 완료 시 전체 산출물 종합 분석 → 평가 초안 생성 |

### 2.11 승인 워크플로우

| # | 기능 | 설명 |
|---|------|------|
| A-D04 | 분석 결과 알림 | AI 분석 완료 시 프로젝트 관리자에게 자동 알림 (완성도 + 요약 포함) |
| A-D05 | 승인 처리 | 관리자가 산출물을 승인 → 단계 완료 / 다음 단계 전환 |
| A-D06 | 보완 요청 처리 | 관리자가 보완 사유 작성 → 팀에 자동 알림 |
| A-D07 | 반려 처리 | 관리자가 반려 사유 작성 → 팀에 재제출 요청 자동 알림 |

### 2.12 자동 독촉

| # | 기능 | 설명 |
|---|------|------|
| A-D08 | 미제출 자동 독촉 | 제출 기한 임박(D-7, D-3) 또는 초과 팀에 자동 알림 발송 |
| A-D09 | 독촉 이력 관리 | 자동 독촉 발송 이력 조회 |
