# USCP 1차년도 기능 목록

> **프로젝트**: 온라인 사회공헌 플랫폼 (USCP)<br/>
> **사업 연도**: 1차년도 (5,000만원 VAT 포함)<br/>
> **작성일**: 2026-04-14

---

## 1. Information Architecture (메뉴 구조도)

### 1.1 공개 영역 (비로그인)

```
USCP
├── 홈
├── 플랫폼 소개
│   ├── USCP란?
│   ├── 참여 방법 안내
│   ├── 캠퍼스별 리빙랩
│   └── FAQ
├── 리빙랩 프로젝트
│   ├── 프로젝트 목록
│   └── 프로젝트 상세
│       ├── 프로젝트 정보
│       ├── 산출물
│       ├── 참여 멤버
│       ├── 참여 기관
│       └── 게시판 (프로젝트별)
├── 지역 문제
│   ├── 이슈 목록
│   └── 이슈 상세
├── 커뮤니티
│   ├── 공지사항
│   ├── 이벤트 공지
│   ├── 자유 게시판
│   ├── 자료실
│   └── 게시글 상세
├── 성공사례
│   ├── 성공사례 목록
│   └── 성공사례 상세
├── 로그인 / 회원가입
├── 비밀번호 찾기
├── 개인정보처리방침
└── 이용약관
```

### 1.2 사용자 영역 (로그인 후)

```
내 정보
├── 프로필 설정
└── 비밀번호 변경
```

### 1.3 관리자 영역

```
관리자 (관리자 역할 접근)
├── 대시보드 (현황 지표)
├── 프로젝트 관리
│   ├── 프로젝트 목록/등록/수정
│   ├── 팀 구성 (팀장 지정, 멤버 배정)
│   └── 산출물 관리
├── 이슈 관리
│   └── 이슈 목록/등록/수정/상태 변경
├── 기관·MOU 관리
│   ├── 기관 목록/등록/수정/삭제
│   └── MOU 등록/상태 관리
├── 커뮤니티 관리
│   ├── 게시판 카테고리 관리
│   └── 게시글 관리 (승인/삭제)
└── 사용자 관리
    ├── 사용자 목록/검색
    ├── 역할 변경
    └── 활성화/비활성화

시스템 설정 (시스템관리자 역할만 접근)
├── 메뉴 관리
├── 역할별 메뉴 접근 설정
└── 감사 로그
    ├── 로그인 이력
    ├── 개인정보 조회 로그
    └── 시스템 활동 로그
```

> **참고**: 본 사업은 **비용 최적화 및 핵심 기능 위주**로 구축됩니다. 소개 페이지(about/guide/campus/FAQ), 성공사례, 배너, 개인정보처리방침·이용약관은 **정적 페이지/이미지**로 제공되며, 관리자가 직접 편집할 수 있는 CMS(콘텐츠 관리 시스템) 기능 및 자동화 기능은 비용 절감을 위해 본 사업 범위에서 제외되었으며, **추후 적용 검토가 필요**합니다.

## 2. 기능 목록

### 2.1 인증·보안

| # | 기능 | 설명 |
|---|------|------|
| S-01 | 회원가입 | 이메일+비밀번호, 이름, 소속, 역할 선택 |
| S-02 | 개인정보 이용 동의 | 회원가입 시 개인정보 수집·이용 동의 체크. 동의 일시 기록 |
| S-03 | 이용약관 동의 | 회원가입 시 서비스 이용약관 동의 체크. 동의 일시 기록 |
| S-04 | 개인정보처리방침 페이지 | 전문 공개 페이지. 정적으로 제공 (편집 기능은 추후 검토) |
| S-05 | 이용약관 페이지 | 전문 공개 페이지. 정적으로 제공 (편집 기능은 추후 검토) |
| S-06 | 로그인 | 이메일+비밀번호 자체 로그인. JWT 토큰 기반 세션 유지 |
| S-07 | 로그아웃 | 세션 종료 및 토큰 무효화 |
| S-08 | 비밀번호 변경 | 로그인 상태에서 현재 비밀번호 확인 후 변경 |
| S-09 | 비밀번호 찾기 | 이메일로 비밀번호 재설정 링크 발송 |
| S-10 | 비밀번호 정책 | 최소 8자, 영문+숫자 조합 필수. 이전 비밀번호 재사용 불가 |

### 2.2 역할·권한 관리

| # | 기능 | 설명 |
|---|------|------|
| R-01 | 시스템 역할 정의 | 시스템관리자 / 관리자 / 교수 / 학생 / 시민 5종 역할 (아래 역할 체계 참조) |
| R-02 | 메뉴 관리 | 시스템 메뉴 목록 관리. 메뉴명, URL, 노출 순서, 상위 메뉴(트리), 아이콘, 활성화 여부 |
| R-03 | 역할별 메뉴 접근 설정 | 역할별로 접근 가능한 메뉴를 설정. 역할×메뉴 매트릭스 체크 화면 |
| R-04 | 역할별 기능 접근 제어 | 역할에 따라 사용 가능한 기능 제한 (이슈 등록=관리자, 산출물 업로드=참여자 등) |
| R-05 | 역할 변경 | 관리자 이상 권한으로 사용자의 역할을 변경 |

**시스템 역할 체계**:

| 역할 | 접근 범위 | 설명 |
|------|----------|------|
| **시스템관리자** | 시스템 설정 + 관리자 전체 + 공개/사용자 | 메뉴 관리, 역할 설정, 감사 로그, 약관 편집 등 시스템 운영 전반. 개발/운영사가 담당 |
| **관리자** | 관리자 메뉴 + 공개/사용자 | 프로젝트/이슈/기관/CMS/커뮤니티/사용자 관리. 센터 실무자가 담당 |
| **교수** | 사용자 메뉴 + 공개 | 프로젝트 참여, 산출물 업로드, 게시판 글 작성 |
| **학생** | 사용자 메뉴 + 공개 | 프로젝트 참여, 산출물 업로드, 게시판 글 작성 |
| **시민** | 사용자 메뉴 + 공개 | 게시판 글 작성, 공개 정보 조회 |

> **시스템관리자 vs 관리자 구분**: 관리자(센터 실무자)는 일상적인 콘텐츠 운영을 담당하고, 시스템관리자는 메뉴 구조, 권한 설정, 감사 로그 등 시스템 레벨 설정을 담당합니다.

### 2.4 감사 로그

| # | 기능 | 설명 |
|---|------|------|
| L-01 | 로그인 이력 | 로그인 성공/실패 일시, IP 주소, 접속 기기 기록 |

### 2.5 리빙랩 프로젝트 관리

| # | 기능 | 설명 |
|---|------|------|
| P-01 | 프로젝트 등록 | 관리자가 생성. 제목, 설명, 기간, 예산, 캠퍼스, SDGs 태그 |

| P-05 | 프로젝트 공개 목록 | 카드 그리드. 상태별/캠퍼스별 필터. 비로그인 접근 |
| P-06 | 프로젝트 공개 상세 | 탭 구성 (프로젝트 정보/산출물/멤버/기관/게시판) |
| P-07 | 관리자 프로젝트 목록 | 테이블 형태. 상태 변경, 필터, 검색 |
| P-08 | 팀 생성 | 프로젝트 내 팀 생성. 팀명, 참여 그룹 지정 |
| P-09 | 팀장 지정 | 팀별 팀장 1명 지정 |
| P-10 | 멤버 배정 | 등록된 사용자를 팀에 배정 |
| P-11 | 멤버 목록 조회 | 프로젝트 상세에서 팀별 멤버 조회 |
| P-12 | 산출물 업로드 | 참여자가 파일 업로드. 제목, 설명, 유형 태그(보고서/데이터/프로토타입/양식 템플릿) |
| P-13 | 산출물 다운로드 | 업로드된 파일 다운로드 |
| P-14 | 산출물 목록 | 프로젝트 상세에서 산출물 목록. 유형별 필터 |
| P-15 | 산출물 삭제 | 업로드한 본인 또는 관리자가 삭제 |
| P-16 | 문서 양식 등록 | 관리자가 양식 템플릿을 "양식 템플릿" 유형으로 업로드 |
| P-17 | 사업화 메모 | 프로젝트 상세에 연계 사업 정보를 텍스트로 기록 |
