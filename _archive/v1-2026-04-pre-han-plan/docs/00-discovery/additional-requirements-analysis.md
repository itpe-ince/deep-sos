## 1. 문헌 핵심 내용 요약

### 1.1 캠퍼스 특성화 (p.31)

- **허브형 캠퍼스**: 대전(글로벌 연구 거점), 공주(지역사회 거점)
- **스포크형 캠퍼스**: 예산·내포, 세종, 신동, 보운, 천안
- **공주캠퍼스 = 지역사회 거점 = 지역사회특화센터 중심**
- 지자체·공공기관과 연계한 교육혁신·연구개발 → 지역사회 난제 해결

### 1.2 2+2 멀티트랙 교육체계 (p.45~48)

- **2(기초강화)**: 1-2학년 (교양기초, 전공기초)
- **2(트랙)**: 3-4학년 (실무인재양성, 학문인재양성)
- 학습지원플랫폼 4종:
  - AI 기반 **Union With U+** (맞춤형 비교과 추천, 포트폴리오)
  - **학생성공센터** (역량별 인증서, 진로/취업 연계)
  - **실무인재양성플랫폼** (지역산업특화센터·지역사회특화센터)
  - **고급인재양성플랫폼** (미래융합학부)

### 1.3 GLOCAL 6대 핵심역량 (p.46)

Global / Local / Open-minded / Creative / AI / Leadership

### 1.4 리빙랩 프로젝트 교과목 운영 (p.49) ⭐

**핵심 운영 절차**:
```
1. 지역현안 제시 (센터·대학)
2. 문제시나리오 설계 + 오리엔테이션 (PBL 시나리오 자문단)
3. 문제해결 과정 (팀별 문제분석 → 퇴함 계획서 → 중간보고 → 최종발표)
4. 중간결과물 평가 (교수자/지역전문가 멘토 평가 + 학생상호평가 + 성찰일지)
5. 우수성과 공유회 (우수방안 채택 및 활용)
```

- **AI 기반 지역문제 수집** → 혁신기관 모집/참여 → 리빙랩 교과목 운영 → 성과 확산

### 1.5 지역특화센터 체계 (p.54~55) ⭐⭐

- **지역사회특화센터** (지역난제해결형) — **USCP 주 대상**
  - 지자체·공공기관 연계 **정책모델형 공동연구**
  - 지역 현안 기반 PBL, 리빙랩, 사회문제 해결 융합트랙 운영
  - 참여 주체: 내·외부 **JA(Joint Appointment) 교원** + 지자체 실무자 + 학생
- **지역산업특화센터** (산업체현안해결형)
  - 산업체 수요 기반 **마이크로디그리, 계약학과**
  - 실증 기술 검증형 공동 R&D
  - 참여 주체: JA 교원 + 기업 실무자 + 학생

## 3. 신규 추가 권장 기능

### 3.1 [높음] 리빙랩 교과목 운영 관리 ⭐

문헌의 핵심 프로세스(p.49)가 현재 USCP에 없습니다. 리빙랩 프로젝트 관리 기능을 **교과목 형태**로 운영하기 위한 확장입니다.

| 기능 | 설명 | 우선순위 |
|------|------|:-------:|
| LL-01 리빙랩 교과목 등록 | 교과목명, 담당 교수(JA 교원), 학기, 수강 인원, 연계 프로젝트 | 1차 필수 |
| LL-02 PBL 시나리오 자문단 관리 | 자문단 등록, 시나리오 리뷰 이력 | 2차 |
| LL-03 오리엔테이션 자료 관리 | 오리엔테이션 자료 업로드, 참여자 출석 | 2차 |
| LL-04 팀별 문제분석 보고서 | 분석 보고서 업로드 + 피드백 | 1차 필수 |
| LL-05 중간보고 / 최종 발표 | 보고서 업로드 + 발표 자료 + 일정 관리 | 1차 필수 |
| LL-06 교수자·멘토 평가 | 교수자/지역전문가 평가 입력 (점수 + 코멘트) | 2차 |
| LL-07 학생 상호평가 | 팀원 상호평가 폼 + 집계 | 2차 |
| LL-08 성찰일지 | 학생 개인 성찰일지 작성·제출 | 2차 |
| LL-09 우수성과 공유회 | 우수 성과 발표회 일정·참여자·영상/자료 관리 | 2차 |
| LL-10 우수방안 채택·활용 기록 | 채택된 해결안 + 실제 지역/정책 적용 추적 | 2차 |

### 3.2 [높음] JA(Joint Appointment) 교원 관리 ⭐

문헌에서 2029년까지 1,000명 확보 목표. 현재 "사용자 관리"와 별개 개념입니다.

| 기능 | 설명 | 우선순위 |
|------|------|:-------:|
| JA-01 JA 교원 등록 | 내·외부 구분 (교원/지자체 실무자/기업 실무자/지역 전문가) | 1차 필수 |
| JA-02 JA 소속/경력 관리 | 소속 기관, 전문 분야, 이력 | 1차 필수 |
| JA-03 JA 위촉 계약 | 위촉 기간, 역할, 활동 분야, 관련 문서 첨부 | 1차 필수 |
| JA-04 JA 활동 실적 | 참여 프로젝트, 강의, 멘토링 이력 자동 집계 | 2차 |
| JA-05 JA 공개 프로필 | 공개 프로필 페이지 (전문 분야 홍보, 프로젝트 참여 이력) | 2차 |

### 3.3 [중간] 성과 지표(KPI) 관리

문헌에 연차별 정량 목표가 명확합니다. 단, 인터뷰에서 "개인별 KPI 불필요"라고 명시했으므로 **기관/프로그램 단위** 집계만 권장.

| 기능 | 설명 | 우선순위 |
|------|------|:-------:|
| KPI-01 지표 등록 | 지표명, 단위, 연도별 목표치 | 2차 |
| KPI-02 실적 입력 | 월별/분기별 실적 수동 입력 | 2차 |
| KPI-03 자동 집계 지표 | 리빙랩 프로젝트 수, 참여 기관 수, JA 교원 수 등 플랫폼 내 데이터 자동 집계 | 2차 |
| KPI-04 목표 대비 달성률 | 연차별 목표값 vs 실적 비교 대시보드 | 2차 |

### 3.4 [중간] 회의록·의사결정 관리

"지역 거버넌스 내재화"(p.55) 요구사항. MOU 관리의 확장입니다.

| 기능 | 설명 | 우선순위 |
|------|------|:-------:|
| GOV-01 협의체 회의록 관리 | 회의 일시, 참석자, 안건, 결정사항 기록 | 2차 |
| GOV-02 의사결정 이력 | 프로젝트/정책별 의사결정 타임라인 | 2차 |
| GOV-03 정책 피드백 구조 | 프로젝트 성과 → 정책 제안 → 반영 여부 추적 | 2차 |

### 3.5 [낮음] 마이크로디그리/융합트랙 관리

문헌에 강조되나 **교과과정 관리는 대학 LMS 영역**입니다. USCP에서 직접 다루기보다 **정보 안내 수준**만 권장.

| 기능 | 설명 | 우선순위 |
|------|------|:-------:|
| MD-01 마이크로디그리 소개 페이지 | 개설 과정 목록, 이수 요건 안내 (CMS 페이지 활용) | 2차 CMS 활용 |
