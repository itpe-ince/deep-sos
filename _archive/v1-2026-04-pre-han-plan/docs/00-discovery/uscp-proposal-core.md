# USCP 플랫폼 구축 사업계획서 (1·2차년도)

> **프로젝트**: 온라인 사회공헌 플랫폼 (USCP, Union Social Contribution Platform)<br/>
> **발주처**: 국립공주대학교 지역사회특화센터<br/>
> **근거 문서**: 2025 글로컬대학 본지정 실행계획서 / 2026-04-13 요구사항 인터뷰<br/>
> **작성일**: 2026-04-21<br/>
> **범위**: 1차년도 핵심 플랫폼 + 2차년도 고도화 · 센터 운영 체계

---

## 1. 사업 개요

### 1.1 사업 목적

국립공주대학교 2025 글로컬대학 본지정 사업의 핵심 과제인 "온라인 사회공헌 플랫폼(USCP)"을 구축합니다. 대전·공주·예산·세종 4개 캠퍼스의 리빙랩 프로젝트를 체계적으로 관리하고, 지역사회 문제 해결 성과를 대외에 공유하는 통합 플랫폼입니다.

### 1.2 사업 배경

2026-04-13 발주처 인터뷰를 통해 아래 사항을 확인하였습니다.

- **운영 주체**: 지역사회특화센터 전담 인력 **1명**이 플랫폼 전체를 운영
- **사용 방식**: 시민이 자유롭게 글을 올리는 바텀업이 아니라, 센터 담당자가 조사 후 등록하는 **탑다운 방식**
- **관리 수준**: 프로세스 모니터링이 아니라 **산출물(결과물) 중심** 관리
- **범위 조정**: 운영 인력·예산을 고려해 봉사활동 관리, 개인별 KPI 등을 범위에서 제외하고, MVP(Minimum Viable Product, 최소 기능 제품) 구현 후 점진적 개선 진행

### 1.3 운영 원칙

향후 모든 설계·구현 의사결정에 우선 적용되는 원칙입니다.

1. **1인 운영 체제**: 센터 전담 인력 1명이 플랫폼 전체 운영. 불필요한 기능 및 수동 작업을 최소화하는 설계 필수
2. **탑다운 운영**: 센터 담당자가 조사·기획 후 등록
3. **산출물 중심**: 프로세스 모니터링이 아닌 결과물 중심 관리
4. **간소화 우선**: 기능 확장보다 운영 효율을 우선

### 1.4 단계별 추진 전략

| 단계 | 포커스 | 핵심 내용 |
|------|------|-----------|
| **1차년도** | 비용 최적화 + 핵심 기능 | 리빙랩 프로젝트 관리, 이슈, 기관·MOU, 커뮤니티, 시스템관리 기반 |
| **2차년도** | 운영 편의성 + 센터 운영 체계 | CMS 신규 구축, JA 교원 관리, 리빙랩 교과목, KPI, 거버넌스 |

## 3. 1차년도 사업 (핵심 플랫폼 구축)

### 3.1 구축 목표

**"비용 최적화 + 핵심 기능 위주"**

센터 관리자 1인이 효율적으로 운영할 수 있는 플랫폼을 목표로, 반드시 필요한 핵심 기능에 집중하여 구축합니다. 콘텐츠 관리 시스템(CMS), 자동화·알림 기능은 2차년도 이후 도입 검토 대상입니다.

### 3.2 주요 구축 범위

| 영역 | 주요 내용 |
|------|----------|
| 회원·인증·역할 | 이메일 자체 로그인, 5종 역할, 감사 로그 |
| 리빙랩 프로젝트 관리 | 프로젝트 CRUD, 팀 구성, 산출물 관리, 3단계 상태 |
| 지역 문제 관리 | 관리자 등록, 프로젝트 연결 |
| 참여 기관·MOU 관리 | 기관 등록, MOU 상태 관리 |
| 커뮤니티 | 공지사항, 이벤트, 자유 게시판, 자료실, 프로젝트별 게시판 |
| 정적 소개 페이지 | about, guide, campus, FAQ, 약관 등 |
| 시스템관리자 기능 | 메뉴 관리, 권한 설정, 감사 로그 |

### 3.3 상세 기능 설명

#### 3.3.1 회원·인증·역할

사용자가 이메일과 비밀번호로 가입·로그인할 수 있으며, 역할(시스템관리자/관리자/교수/학생/시민)에 따라 접근 가능한 메뉴와 기능이 자동으로 구분됩니다. 회원가입 시 개인정보 수집·이용 동의와 이용약관 동의를 체크하며 동의 일시가 기록됩니다. 비밀번호 정책(최소 8자, 영문+숫자 조합)과 비밀번호 찾기(이메일 재설정 링크 발송) 기능을 포함합니다.

> 세부 기능 목록: **부록 A** 참조

#### 3.3.2 역할·권한 관리 (시스템관리자 전용)

시스템 메뉴를 트리 구조로 관리(메뉴명, URL, 순서, 상위 메뉴, 아이콘, 활성화)하며, 역할별로 접근 가능한 메뉴를 매트릭스 형태로 설정할 수 있습니다. 관리자가 사용자의 역할을 변경하면 접근 권한이 즉시 반영됩니다.

#### 3.3.3 사용자 관리

사용자가 자신의 프로필(이름, 소속, 연락처)을 수정할 수 있습니다. 관리자는 전체 사용자 목록을 검색·조회하고 역할 변경, 활성화/비활성화를 수행합니다. 사용자는 본인 계정을 직접 탈퇴할 수 있으며 개인정보는 삭제 처리됩니다.

#### 3.3.4 감사 로그

법적 요구사항과 개인정보 보호를 위해 주요 활동 이력을 자동 기록합니다. 로그인 성공·실패 이력(IP, 접속 기기), 관리자가 사용자 개인정보를 조회한 이력, 시스템 주요 작업(프로젝트 생성/삭제, 역할 변경, 콘텐츠 발행 등)을 모두 기록하며, 관리자는 각종 로그를 기간·유형·사용자별로 검색·조회할 수 있습니다. 로그는 최소 1년 보관 후 자동 아카이브됩니다.

#### 3.3.5 리빙랩 프로젝트 관리 (핵심 기능)

플랫폼의 가장 핵심 기능입니다. 관리자가 프로젝트를 생성하고(제목, 설명, 수행 기간, 예산, 캠퍼스, SDGs 연계 목표), **발주 → 진행중 → 완료** 3단계로 진행 상태를 관리합니다. 3개 참여 그룹(학생, 지역시민, 교수) 각각에 팀을 구성하고 팀장을 지정합니다.

참여자는 결과물(보고서, 데이터, 프로토타입 등)을 파일로 업로드하며, 관리자가 등록한 양식 템플릿을 다운로드받아 작성 후 업로드하는 흐름을 지원합니다. 최종 산출물 업로드 시 프로젝트 완료 처리가 가능합니다.

프로젝트가 외부 사업(창업, 정책 반영 등)으로 연결된 경우 연계 정보를 텍스트로 기록할 수 있습니다.

비로그인 사용자는 프로젝트 공개 목록(카드 그리드, 상태별 필터)과 상세 페이지(정보·산출물·멤버·기관 탭)를 조회할 수 있으며, 관리자는 전용 관리 화면에서 전체 프로젝트를 테이블로 조회·관리합니다.

#### 3.3.6 지역 문제 관리

센터 담당자가 지역 현안을 조사한 후 직접 등록하는 탑다운 방식입니다(제목, 설명, 카테고리, 캠퍼스, 현장 사진). 비로그인 사용자도 이슈 목록과 상세를 조회할 수 있으며, 이슈에서 관련 리빙랩 프로젝트로 연결이 가능합니다. 관리자는 이슈 상태(등록 / 검토중 / 해결중 / 해결완료 / 보류)를 변경합니다.

#### 3.3.7 참여 기관 / MOU 관리

리빙랩에 참여하는 지자체, 연구기관, 기업, NGO, 대학의 정보를 관리합니다. 기관별로 체결된 MOU(협약 제목, 시작일/종료일, 상태)를 등록하며, 기관을 특정 프로젝트에 연결하여 어떤 기관이 어떤 프로젝트에 참여하는지 확인할 수 있습니다.

#### 3.3.8 커뮤니티 (게시판)

플랫폼 이용자 간 소통과 정보 공유를 위해 5종 게시판을 제공합니다.

- **공지사항**: 관리자 공식 공지 (상단 고정, 중요 표시)
- **이벤트 공지**: 리빙랩 행사, 설명회, 워크숍 (일시/장소/대상/첨부파일)
- **자유 게시판**: 로그인 사용자 자유 작성 (댓글)
- **자료실**: 가이드·양식·보고서 등 파일 중심 (카테고리별 분류, 다운로드 카운트)
- **프로젝트별 게시판**: 프로젝트에 종속된 게시판 (프로젝트 참여자만 작성, 프로젝트 상세 탭에서 접근)

모든 게시판은 WYSIWYG 작성, 파일 첨부(복수), 댓글, 검색, 정렬을 지원합니다.

#### 3.3.9 정적 소개 페이지

본 사업은 비용 최적화 방침에 따라 대외 홍보용 소개 페이지(플랫폼 소개, 참여 방법, 캠퍼스별 리빙랩, FAQ)와 메인 배너, 개인정보처리방침, 이용약관을 **정적 방식**으로 제공합니다. 최초 콘텐츠는 발주처 제공 원고를 기반으로 개발 단계에 반영하며, 이후 콘텐츠 변경이 필요한 경우 유지보수 지원으로 처리합니다. 관리자가 직접 편집하는 CMS 기능은 2차년도에 구축됩니다.

#### 3.3.10 메인 홈 / 관리자 대시보드

메인 홈은 히어로 배너, 주요 지표(프로젝트 수/이슈 수/참여 기관 수/참여자 수), 최근 등록된 프로젝트와 이슈, 4개 캠퍼스 소개를 표시합니다. 관리자 대시보드는 전체 현황 수치와 최근 등록 항목 목록을 제공합니다.

## 4. 2차년도 사업 (고도화 + 센터 운영 체계)

### 4.1 구축 목표

**"운영 편의성 강화 + 지역사회특화센터 운영 체계 구축"**

1차년도에 정적으로 제공되던 소개 페이지를 **CMS로 편집 가능**하게 전환하고, 자동화·시각화·알림으로 관리자 운영 부담을 낮춥니다. 실행계획서에 명시된 **JA 교원 관리, 리빙랩 교과목 운영, KPI, 거버넌스** 등 센터의 실제 운영 업무를 체계적으로 지원하는 기능을 신규로 구축합니다.

### 4.2 주요 구축 범위

| 파트 | 주요 내용 |
|------|----------|
| **A. 운영 편의성 강화** | 소셜 로그인, 프로젝트 권한, CMS 신규 구축, 알림, 대시보드 차트 |
| **B. 지역사회특화센터 운영 체계** | JA 교원, 리빙랩 교과목, KPI, 거버넌스 |
| **C. 시스템 설정 관리** | 하드코딩된 설정을 UI에서 관리 |

### 4.3 상세 기능 설명

#### 4.3.1 소셜 로그인 추가

1차년도의 이메일 자체 로그인에 소셜 로그인(카카오/네이버/구글)을 추가합니다. 카카오/네이버/구글 계정으로 별도 회원가입 없이 자동 회원 생성되며, 기존 이메일 계정에 소셜 계정을 연결하여 복수 로그인 수단을 제공합니다.

#### 4.3.2 프로젝트 권한 관리

1차년도에 시스템 관리자만 프로젝트를 관리했던 것을 확장하여, 프로젝트 참여자가 직접 관리할 수 있도록 **4단계 프로젝트 역할**을 부여합니다.

- **프로젝트 관리자**: 프로젝트 설정, 팀 구성, 상태 전환 (교수 등)
- **팀장**: 팀 멤버 관리, 산출물 등록/삭제
- **팀원**: 산출물 등록, 게시판 글 작성
- **열람자**: 공개 정보 조회

#### 4.3.3 이슈·기관 관리 고도화

이슈 상세 페이지에서 **원클릭**으로 해당 이슈를 기반으로 프로젝트를 자동 생성할 수 있습니다. 이슈 상태 변경 시 이력이 타임라인으로 자동 기록되며, 관리자 화면에서 카테고리별·상태별 통계 차트를 조회할 수 있습니다.

MOU 종료일 30일 전에 관리자에게 자동 알림이 발송되고, 기관 상세 페이지에서 해당 기관이 참여 중인 프로젝트 목록을 자동 조회할 수 있습니다. 사업화 연계 정보는 단순 텍스트 메모에서 구조화된 이력 관리로 확장됩니다.

#### 4.3.4 CMS 신규 구축 (콘텐츠 관리)

1차년도에 정적으로 제공되던 소개 페이지, 배너, 성공사례, 약관을 **관리자가 직접 편집할 수 있는 CMS 체계**로 신규 구축합니다. 발주처가 개발사에 의존하지 않고 콘텐츠를 자체 운영할 수 있게 되는 핵심 고도화입니다.

- **소개 페이지**: 스마트 에디터로 편집 (기본 서식, 파일/이미지/YouTube/이모지 + 동영상 직접 업로드, SNS 임베드, 지도, 표 편집, 드래그 앤 드롭 단락 순서 조절). 발행 예약, 수정 이력 자동 기록
- **배너**: 등록/수정/순서 변경, 시작일/종료일 설정 시 자동 노출·숨김
- **성공사례**: 전용 관리 화면 + **4단계 스토리**(문제 → 과정 → 결과 → 영향) + 정책 반영 기록
- **약관**: 개인정보처리방침·이용약관을 관리자가 편집, 버전 관리 (시행일 기록), 주요 변경 시 사용자에게 재동의 요청

#### 4.3.5 관리자 대시보드 고도화

1차년도 텍스트 수치를 **시각화 차트**(프로젝트 현황, 캠퍼스별 비교, 월별 추이)로 전환합니다. 미처리 산출물, 만료 임박 MOU 등 미처리 항목 수를 뱃지로 표시하여 관리자가 우선순위를 쉽게 파악할 수 있게 합니다.

#### 4.3.6 실시간 알림 시스템

Server-Sent Events(SSE) 기반 실시간 알림 시스템을 구축합니다. 헤더의 벨 아이콘에 새 알림 수가 표시되며, 드롭다운에서 최근 알림을 확인·읽음 처리할 수 있습니다. 알림 대상은 산출물 등록, MOU 만료 임박, 이슈 상태 변경 등입니다.

#### 4.3.7 JA(Joint Appointment) 교원 관리

2025 글로컬대학 실행계획서에 명시된 **2029년까지 JA 교원 1,000명 확보 목표**에 대응하는 기능입니다. 내·외부 JA 교원(교원/지자체 실무자/기업 실무자/지역 전문가)을 체계적으로 관리합니다. 교원별 소속 기관·전문 분야·위촉 계약(기간·역할·활동 분야·관련 문서)을 등록하고, 참여 프로젝트·멘토링 이력이 자동 집계됩니다. JA 교원의 전문 분야와 참여 이력은 외부에 공개 프로필로 노출할 수 있습니다.

#### 4.3.8 리빙랩 교과목 운영

실행계획서 p.49 "지역문제해결을 위한 리빙랩 프로젝트 교과목 운영"에 대응하는 기능입니다. 리빙랩 프로젝트가 단순 팀 활동이 아니라 **정식 교과목(학점 연계)** 으로 운영되는 경우를 지원합니다.

교과목 정보(교과목명, 담당 교수, 학기, 수강 인원, 연계 프로젝트)를 등록하고, 팀별 문제분석 보고서 → 중간보고 → 최종 발표 단계별 산출물을 관리합니다. 교수자·지역전문가 멘토 평가, 학생 상호평가, 성찰일지를 모두 플랫폼에서 처리하며, 우수 성과를 공유회에서 발표하고 실제 지역/정책에 반영된 내역을 추적할 수 있습니다.

#### 4.3.9 성과 지표(KPI) 관리

실행계획서의 연차별 정량 목표에 대응합니다. 지표별 연도별 목표치(예: 융합과정 이수율, JA 교원 수, 마이크로디그리 과정 수)를 등록하고, 월별·분기별 실적을 입력합니다. 플랫폼 내 데이터(프로젝트 수, 기관 수, JA 교원 수)는 자동 집계되며, 목표 대비 달성률을 대시보드에서 한눈에 확인합니다.

> 개인별 KPI는 개인정보 이슈로 제외하고, 기관·프로그램 단위 집계만 수행합니다.

#### 4.3.10 거버넌스 (회의록·의사결정)

실행계획서 "지역 거버넌스 내재화"에 대응합니다. 협의체 회의록(일시·참석자·안건·결정사항)을 등록하고, 프로젝트·정책별 의사결정 타임라인을 관리합니다. 프로젝트 성과가 정책에 반영되는 과정을 추적(성과 → 정책 제안 → 반영 여부)합니다.

#### 4.3.11 시스템 설정 관리 (시스템관리자 전용)

1차년도에서 하드코딩·환경변수로 관리하던 설정을 **UI에서 체계적으로 관리**할 수 있도록 확장합니다. 발주처가 개발사에 의존하지 않고 시스템의 기본 구성을 조정할 수 있게 됩니다.

- **사이트 기본 정보**: 사이트명, 로고, 대표 이메일, 기관 정보
- **이메일 발송 설정**: SMTP 서버, 발신자
- **파일 업로드 설정**: 허용 확장자, 최대 용량
- **보안 설정**: 세션 타임아웃, 로그인 실패 제한, 비밀번호 정책
- **알림 설정**: 알림 유형별 활성화, 발송 채널
- **캠퍼스 관리**: 캠퍼스 목록, 순서, 활성화
- **환경 변수 관리**: 외부 API 키, 시스템 파라미터 (재시작 없이 적용)

| R-02 | 메뉴 관리 | 시스템 메뉴 목록 관리. 메뉴명, URL, 노출 순서, 상위 메뉴(트리), 아이콘, 활성화 여부 |
| R-03 | 역할별 메뉴 접근 설정 | 역할별로 접근 가능한 메뉴를 설정. 역할×메뉴 매트릭스 체크 화면 |
| R-04 | 역할별 기능 접근 제어 | 역할에 따라 사용 가능한 기능 제한 (이슈 등록=관리자, 산출물 업로드=참여자 등) |
| R-05 | 역할 변경 | 관리자 이상 권한으로 사용자의 역할을 변경 |

**시스템 역할 체계**:

| 역할 | 접근 범위 | 설명 |
|------|----------|------|
| **시스템관리자** | 시스템 설정 + 관리자 전체 + 공개/사용자 | 메뉴 관리, 역할 설정, 감사 로그, 약관 편집 등 시스템 운영 전반. 개발/운영사가 담당 |
| **관리자** | 관리자 메뉴 + 공개/사용자 | 프로젝트/이슈/기관/CMS/커뮤니티/사용자 관리. 센터 실무자가 담당 |
| **교수** | 사용자 메뉴 + 공개 | 프로젝트 참여, 산출물 업로드, 게시판 글 작성 |
| **학생** | 사용자 메뉴 + 공개 | 프로젝트 참여, 산출물 업로드, 게시판 글 작성 |
| **시민** | 사용자 메뉴 + 공개 | 게시판 글 작성, 공개 정보 조회 |

> **시스템관리자 vs 관리자 구분**: 관리자(센터 실무자)는 일상적인 콘텐츠 운영을 담당하고, 시스템관리자는 메뉴 구조, 권한 설정, 감사 로그 등 시스템 레벨 설정을 담당합니다.

#### 리빙랩 교과목 운영

> 실행계획서 p.49 "지역문제해결을 위한 리빙랩 프로젝트 교과목 운영"에 대응. 리빙랩 프로젝트가 정식 교과목(학점 연계)으로 운영되는 경우를 지원합니다.

| # | 기능 | 설명 |
|---|------|------|
| LL-01 | 리빙랩 교과목 등록 | 교과목명, 담당 교수(JA 교원), 학기, 수강 인원, 연계 프로젝트 |
| LL-02 | 팀별 문제분석 보고서 | 분석 보고서 업로드 + 피드백 기능 |
| LL-03 | 중간보고 / 최종 발표 | 보고서 업로드 + 발표 자료 + 일정 관리 |
| LL-04 | 교수자·멘토 평가 | 교수자/지역전문가 평가 입력 (점수 + 코멘트) |
| LL-05 | 학생 상호평가 | 팀원 상호평가 폼 + 집계 |
| LL-06 | 성찰일지 | 학생 개인 성찰일지 작성·제출 |
| LL-07 | 우수성과 공유회 | 우수 성과 발표회 일정·참여자·자료 관리 |
| LL-08 | 우수방안 채택·활용 기록 | 채택된 해결안 + 실제 지역/정책 적용 추적 |
