## 2. 발주처 RFI 이해도

### 2.1 사업 배경 파악

발주처는 **2025 글로컬대학 본지정 사업**의 핵심 과제로 USCP 플랫폼을 구축하고자 하며, 다음의 제약 조건이 확인되었습니다.

- **운영 주체**: 지역사회특화센터 전담 인력 **1명**이 플랫폼 전체 운영
- **사용 방식**: 센터 담당자가 조사 후 등록하는 **탑다운 방식** (시민 자유 등록 아님)
- **관리 수준**: 프로세스 모니터링이 아닌 **산출물(결과물) 중심** 관리
- **범위 조정**: 운영 인력·예산 고려 시 MVP 우선 구현 후 점진적 개선 필요

### 2.2 핵심 요구사항 도출

| 요구사항 분류 | 내용 |
|--------------|------|
| **대상 캠퍼스** | 대전·공주·예산·세종 4개 캠퍼스 리빙랩 |
| **사용자 그룹** | 시스템관리자·관리자·교수·학생·시민 (5종 역할) |
| **1차 핵심 기능** | 리빙랩 프로젝트 관리 · 지역문제 관리 · 기관/MOU 관리 · 커뮤니티 |
| **2차 고도화** | CMS 신규 구축 · JA 교원 · 리빙랩 교과목 · KPI · 거버넌스 · 시스템 설정 |
| **정량 목표 (실행계획서)** | 2029년까지 JA 교원 **1,000명** 확보 · 연간 수백 건의 산출물 처리 |

### 2.3 주요 도전 과제 (당사 진단)

1. **1인 운영 vs. 1,000명 규모 관리 — 본질적 불균형**
   → 수동 업무의 시스템 이관 + 정교한 권한·알림 설계 필요
2. **단계별 예산 제약 — 우선순위 관리 필요**
   → 1차년도에 "정적 제공 + 운영 기반" 구축, 2차에 CMS·센터 체계로 단계적 전개
3. **실행계획서 기반 정량 KPI 추적**
   → JA 교원 수, 융합과정 이수율 등 지표를 플랫폼 데이터로 자동 집계하는 구조 설계 필요

## 3. 당사 제안 방향 (요약)

### 3.1 구축 철학

> **"1인 운영을 가능하게 하는 플랫폼" — 수동 업무를 시스템으로, 운영자는 의사결정에 집중**

- **간소화 우선**: 기능 확장보다 운영 효율 우선
- **데이터 자산화**: 1차년도 데이터 구조가 2차 운영 체계의 기반이 됨
- **단계적 편의성 확보**: 1차 시스템화 → 2차 CMS·알림·자동화로 부담 경감

### 3.2 단계별 추진 전략

| 단계 | 기간(제안) | 포커스 | 핵심 내용 |
|------|:-------:|--------|----------|
| **1차년도** | 8개월 | 비용 최적화 + 핵심 기능 | 리빙랩 프로젝트·이슈·기관/MOU·커뮤니티·시스템관리 기반 |
| **2차년도** | 5개월 | 운영 편의성 + 센터 운영 체계 | CMS 신규 · JA 교원 · 리빙랩 교과목 · KPI · 거버넌스 |

### 3.3 단계별 데이터 흐름 (논리)

```
1차년도: 운영 데이터 축적 (프로젝트·산출물·이슈·MOU)
        ↓
2차년도: 데이터 구조화 + 편집 가능 (CMS·KPI·JA·거버넌스)
```

> **참고**: AI 기반 업무 자동화는 데이터 축적 후 별도 제안으로 제시 가능하며, 본 답변서에는 포함되지 않았습니다.

**"비용 최적화 + 핵심 기능 위주"**

**"운영 편의성 강화 + 지역사회특화센터 운영 체계 구축"**

1차년도의 정적 소개 페이지를 **CMS 편집 가능**하게 전환하고, 자동화·시각화·알림으로 관리자 운영 부담을 낮춥니다. 실행계획서에 명시된 **JA 교원 관리, 리빙랩 교과목 운영, KPI, 거버넌스**를 신규로 구축합니다.

| **B. 지역사회특화센터 운영 체계** | JA 교원, 리빙랩 교과목, KPI, 거버넌스 |
